성능 맵 지원¶

지원되는 플랫폼(Linux 및 macOS)에서 런타임은 perf map 파일 을 활용하여 Python 함수를 외부 프로파일링 도구(예: perf 또는 samply)가 볼 수 있도록 할 수 있습니다. 실행 중인 프로세스는 /tmp 디렉토리에 파일을 생성할 수 있으며, 이 파일은 실행 파일 코드의 섹션을 이름에 매핑할 수 있는 항목을 포함합니다. 이 인터페이스는 Linux Perf 도구 <https://git.kernel.org/pub/scm/linux/ kernel/git/torvalds/linux.git/tree/tools/perf/Documentation/jit-interface.txt> 의 문서를 참조하십시오.

Python에서는 이러한 헬퍼 API를 라이브러리 및 기능을 통해 사용할 수 있습니다. 이들은 온디바이에 기계 코드를 생성하는 데 의존합니다.

Note that holding an attached thread state is not required for these APIs.

int PyUnstable_PerfMapState_Init(void)¶
이것은 불안정 API. 경고 없이 사소 버전에서 변경될 수 있습니다.

PyUnstable_WritePerfMapEntry() 를 통해 쓰기는 스레드 안전성을 보장하기 위해 (이미 열려 있지 않은 경우) /tmp/perf-$pid.map 파일을 열고 락을 생성해야 합니다. 일반적으로 명시적으로 호출할 필요가 없습니다. PyUnstable_WritePerfMapEntry() 를 사용하기만 하면 첫 호출 시 상태가 초기화됩니다.

성공하면 0 을 반환하고, perf 맵 파일을 생성/열지 못할 경우 -1, 락을 생성하지 못할 경우 -2 를 반환합니다. 실패 원인에 대한 더 많은 정보는 errno 를 확인하십시오.

int PyUnstable_WritePerfMapEntry(const void *code_addr, size_t code_size, const char *entry_name)¶
이것은 불안정 API. 경고 없이 사소 버전에서 변경될 수 있습니다.

/tmp/perf-$pid.map 파일에 하나의 항목을 작성합니다. 이 함수는 스레드 안전합니다. 예시 항목은 다음과 같습니다.

# 주소      크기  이름
7f3529fcf759 b     py::bar:/run/t.py

perf 맵 파일이 이미 열려 있지 않은 경우, PyUnstable_PerfMapState_Init() 를 호출하여 항목을 작성하기 전에, 다음을 반환합니다. 성공하면 0 을, 실패하면 PyUnstable_PerfMapState_Init() 와 동일한 오류 코드를 반환합니다.

void PyUnstable_PerfMapState_Fini(void)¶
이것은 불안정 API. 경고 없이 사소 버전에서 변경될 수 있습니다.

:c:func:`PyUnstable_PerfMapState_Init`에 의해 열린 perf 맵 파일을 닫습니다. 이는 인터프리터 종료 중 런타임 자체에 의해 호출됩니다. 일반적으로 포킹(forking)과 같은 특정 시나리오를 처리하는 경우를 제외하고는 명시적으로 호출할 이유가 없습니다.

int PyUnstable_CopyPerfMapFile(const char *parent_filename)¶
이것은 불안정 API. 경고 없이 사소 버전에서 변경될 수 있습니다.

/tmp/perf-$pid.map 파일을 열고 parent_filename 의 내용을 파일에 추가합니다.

이 함수는 모든 플랫폼에서 사용할 수 있지만 perf 맵을 지원하는 플랫폼(현재는 Linux만 해당)에서만 출력을 생성합니다. 다른 플랫폼에서는 아무 작업도 하지 않습니다.

Added in version 3.13.

int PyUnstable_PerfTrampoline_CompileCode(PyCodeObject *code)¶
이것은 불안정 API. 경고 없이 사소 버전에서 변경될 수 있습니다.

현재 perf 트램폴린을 사용하여 주어진 코드 객체를 컴파일합니다.

“현재” 트램폴린은 런타임에 의해 설정되거나 가장 최근의 PyUnstable_PerfTrampoline_SetPersistAfterFork() 호출에 의해 설정된 트램폴린입니다.

트램폴린이 설정되지 않으면, 일반 컴파일(perf 맵 항목 없음)으로 폴백합니다.

매개변수:

  • code – 컴파일할 코드 객체입니다.

반환:

성공 시 0, 실패 시 -1.

Added in version 3.13.

int PyUnstable_PerfTrampoline_SetPersistAfterFork(int enable)¶
이것은 불안정 API. 경고 없이 사소 버전에서 변경될 수 있습니다.

perf 트램폴린이 포크 이후에도 지속할지 여부를 설정합니다.

  • enable 이 true인 경우(0이 아닌 값): perf 맵 파일이 포크 후에도 열려 있거나 유효합니다. 자식 프로세스는 기존의 모든 perf 맵 항목을 상속받습니다.

  • enable 이 false인 경우(0): perf 맵이 포크 후 닫힙니다. 자식 프로세스는 빈 perf 맵을 받습니다.

기본값: false (포크 시 지워짐).

매개변수:

  • enable – 1은 활성화, 0은 비활성화입니다.

반환:

성공 시 0, 실패 시 -1.

Added in version 3.13.
