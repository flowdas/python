C API 모니터링¶

버전 3.13에 추가됨.

확장 기능은 이벤트 모니터링 시스템과 상호 작용해야 할 수 있습니다. 이벤트 구독 및 콜백 등록은 :mod:`sys.monitoring`에 노출된 Python API를 통해 수행할 수 있습니다.

실행 이벤트 생성¶

아래 함수들은 확장 기능이 Python 코드 실행을 에뮬레이션하는 동안 모니터링 이벤트를 발생시킬 수 있도록 합니다. 각 함수는 이벤트 활성화 상태에 대한 간결한 정보와, 코드 객체를 나타내는 PyObject*, 명령어 오프셋, 그리고 때로는 추가적인 이벤트별 인자(다양한 이벤트 콜백의 서명에 대한 자세한 내용은 sys.monitoring 참고)를 포함하는 PyMonitoringState 구조체를 이벤트를 받습니다. codelike 인수는 :class:`types.CodeType`의 인스턴스이거나 이를 에뮬레이션하는 유형이어야 합니다.

VM은 이벤트를 발생시킬 때 추적을 비활성화하므로, 사용자 코드가 그럴 필요가 없습니다.

모니터링 함수는 현재 예외를 다루는 다음 함수들을 제외하고는 예외가 설정된 상태로 호출해서는 안 됩니다.

type PyMonitoringState¶

이벤트 유형의 상태 표현입니다. 내용물은 아래 설명된 모니터링 API 함수에 의해 유지되며, 사용자가 할당합니다.

아래의 모든 함수들은 성공 시 0을, 오류 시 -1(예외 설정과 함께)을 반환합니다.

이벤트 설명은 :mod:`sys.monitoring`를 참조하십시오.

모니터링 상태 관리¶

모니터링 상태는 모니터링 스코프를 이용해 관리할 수 있습니다. 스코프는 일반적으로 Python 함수에 해당합니다.

int PyMonitoring_EnterScope(PyMonitoringState *state_array, uint64_t *version, const uint8_t *event_types, Py_ssize_t length)¶

모니터링 스코프에 진입합니다. event_types 는 스코프에서 발생할 수 있는 이벤트 ID 배열입니다. 예를 들어, PY_START 이벤트의 ID는 PY_MONITORING_EVENT_PY_START 값이며, 이는 sys.monitoring.events.PY_START 의 2의 로그 값과 숫자적으로 동일합니다. state_array 는 event_types 의 각 이벤트에 대한 모니터링 상태 항목을 가진 배열입니다. 이 배열은 사용자에 의해 할당되지만, PyMonitoring_EnterScope() 에 의해 이벤트의 활성화 상태에 대한 정보로 채워집니다. event_types 의 크기(따라서 state_array 의 크기)는 length 에 있습니다.

version 인수는 사용자가 state_array 와 함께 할당하고 0으로 초기화해야 하는 값에 대한 포인터이며, 오직 PyMonitoring_EnterScope() 자체에 의해서만 설정됩니다. 이를 통해 해당 함수는 이전 호출 이후 이벤트 상태가 변경되었는지 여부를 판단할 수 있으며, 변경되지 않았다면 빠르게 반환할 수 있습니다.

여기서 언급하는 스코프는 어휘적 스코프입니다: 함수, 클래스 또는 메서드입니다. PyMonitoring_EnterScope() 는 어휘적 스코프에 진입할 때 호출해야 합니다. 스코프는 재진입될 수 있으며, 재귀적인 Python 함수를 에뮬레이션하는 상황과 같이 동일한 state_array 와 version 을 재사용합니다. 제너레이터를 에뮬레이션하는 경우와 같이 코드 실행이 일시 중지되면, 스코프를 종료하고 다시 진입해야 합니다.
