프로파일링 및 추적¶

파이썬 인터프리터는 프로파일링 및 실행 추적 기능을 연결하기 위한 일부 저수준 지원을 제공합니다. 이는 프로파일링, 디버깅 및 커버리지 분석 도구에서 사용됩니다.

이 C 인터페이스를 사용하면 프로파일링 또는 추적 코드가 Python 레벨 콜러블 객체 호출의 오버헤드를 피하고 대신 직접 C 함수 호출을 할 수 있습니다. 해당 기능의 본질적인 속성은 변경되지 않았습니다. 이 인터페이스는 스레드별로 추적 함수를 설치할 수 있도록 하며, 추적 함수에 보고되는 기본 이벤트는 이전 버전의 Python 레벨 추적 함수에 보고되었던 것과 동일합니다.

typedef int (*Py_tracefunc)(PyObject *obj, PyFrameObject *frame, int what, PyObject *arg)¶

PyEval_SetProfile() 및 PyEval_SetTrace() 를 사용하여 등록된 추적 함수의 타입입니다. 첫 번째 매개변수는 등록 함수에 obj 로 전달되는 객체이며, frame 은 이벤트가 속한 프레임 객체이고, what 은 PyTrace_CALL, PyTrace_EXCEPTION, PyTrace_LINE, PyTrace_RETURN, PyTrace_C_CALL, PyTrace_C_EXCEPTION, PyTrace_C_RETURN, 또는 PyTrace_OPCODE 상수 중 하나이고, arg 는 what 의 값에 따라 달라집니다:

what 의 값

arg 의 의미

PyTrace_CALL

항상 :c:data:`Py_None`입니다.

PyTrace_EXCEPTION

sys.exc_info()\에 의해 반환되는 예외 정보입니다.

PyTrace_LINE

항상 :c:data:`Py_None`입니다.

PyTrace_RETURN

호출자에게 반환되는 값, 또는 예외로 인해 발생한 경우 `NULL`입니다.

PyTrace_C_CALL

호출되는 함수 객체.

PyTrace_C_EXCEPTION

호출되는 함수 객체.

PyTrace_C_RETURN

호출되는 함수 객체.

PyTrace_OPCODE

항상 :c:data:`Py_None`입니다.
int PyTrace_CALL¶

함수나 메서드에 대한 새 호출이 보고되거나, 제너레이터에 대한 새 항목이 보고될 때 Py_tracefunc 함수에 대한 what 매개 변수의 값. 제너레이터 함수에 대한 이터레이터 생성은 해당 프레임의 파이썬 바이트 코드로의 제어 전달이 없기 때문에 보고되지 않음에 유의하십시오.

int PyTrace_EXCEPTION¶

예외가 발생했을 때 Py_tracefunc 함수에 대한 what 매개 변수의 값. 콜백 함수는 실행되는 프레임 내에서 바이트 코드가 처리된 후 예외가 설정될 때 what 에 대해 이 값으로 호출됩니다. 이것의 효과는 예외 전파로 인해 파이썬 스택이 되감기는 것인데, 예외가 전파되어 각 프레임으로 반환될 때 콜백이 호출됩니다. 추적 함수만 이러한 이벤트를 받습니다. 프로파일러에게는 필요하지 않습니다.

int PyTrace_LINE¶

라인 번호 이벤트가 보고될 때 Py_tracefunc 함수에 전달되는 what 매개 변수의 값 (하지만 프로파일링 함수는 제외). 해당 프레임에서 f_trace_lines 를 0 으로 설정하여 비활성화할 수 있습니다.

int PyTrace_RETURN¶

호출이 반환되려고 할 때 Py_tracefunc 함수에 대한 what 매개 변수의 값.

int PyTrace_C_CALL¶

C 함수가 호출되려고 할 때 Py_tracefunc 함수에 대한 what 매개 변수의 값.

int PyTrace_C_EXCEPTION¶

C 함수에서 예외가 발생했을 때 Py_tracefunc 함수에 대한 what 매개 변수의 값.

int PyTrace_C_RETURN¶

C 함수가 반환되었을 때 Py_tracefunc 함수에 대한 what 매개 변수의 값.

int PyTrace_OPCODE¶

새로운 옵코드가 실행되려고 할 때 Py_tracefunc 함수에 대한 what 매개 변수의 값 (하지만 프로파일링 함수는 제외). 이 이벤트는 기본적으로 방출되지 않습니다. 해당 프레임에서 f_trace_opcodes 를 1 로 설정하여 명시적으로 요청해야 합니다.

void PyEval_SetProfile(Py_tracefunc func, PyObject *obj)¶

프로파일러 함수를 func*로 설정합니다. *obj 매개 변수는 함수의 첫 번째 매개 변수로 전달될 수 있으며, 모든 파이썬 객체 또는 NULL`일 수 있습니다. 프로파일 함수가 상태를 유지해야 하는 경우, 각 스레드에 대해 다른 *obj* 값을 사용하는 것은 저장하기 편리하고 스레드 안전한 공간을 제공합니다. 프로파일 함수는 :c:data:`PyTrace_LINE 및 :c:data:`PyTrace_OPCODE`와 :c:data:`PyTrace_EXCEPTION`을 제외한 모든 모니터링된 이벤트에 대해 호출됩니다.

또한 sys.setprofile() 함수를 참조하십시오.

호출자는 :term:`첨부된 스레드 상태 <attached thread state>`를 가지고 있어야 합니다.

void PyEval_SetProfileAllThreads(Py_tracefunc func, PyObject *obj)¶

:c:func:`PyEval_SetProfile`와 유사하지만, 현재 인터프리터에 속한 실행 중인 모든 스레드에 프로파일 함수를 설정하며 현재 스레드에만 설정하는 것을 제외합니다.

호출자는 :term:`첨부된 스레드 상태 <attached thread state>`를 가지고 있어야 합니다.

:c:func:`PyEval_SetProfile`와 같이, 이 함수는 모든 스레드에서 프로파일 함수를 설정하는 동안 발생하는 모든 예외를 무시합니다.

Added in version 3.12.

void PyEval_SetTrace(Py_tracefunc func, PyObject *obj)¶

추적 함수를 func*로 설정합니다. 이것은 :c:func:`PyEval_SetProfile`와 유사하지만, 추적 함수는 라인 번호 이벤트와 옵코드 이벤트를 받지만, C 함수 객체 호출과 관련된 이벤트는 받지 않습니다. :c:func:`PyEval_SetTrace`를 사용하여 등록된 모든 추적 함수는 *what 매개 변수의 값으로 PyTrace_C_CALL, PyTrace_C_EXCEPTION 또는 :c:data:`PyTrace_C_RETURN`을 받지 못합니다.

또한 sys.settrace() 함수를 참조하십시오.

호출자는 :term:`첨부된 스레드 상태 <attached thread state>`를 가지고 있어야 합니다.

void PyEval_SetTraceAllThreads(Py_tracefunc func, PyObject *obj)¶

:c:func:`PyEval_SetTrace`와 유사하지만, 현재 인터프리터에 속한 실행 중인 모든 스레드에 추적 함수를 설정하며 현재 스레드에만 설정하는 것을 제외합니다.

호출자는 :term:`첨부된 스레드 상태 <attached thread state>`를 가지고 있어야 합니다.

:c:func:`PyEval_SetTrace`와 같이, 이 함수는 모든 스레드에서 추적 함수를 설정하는 동안 발생하는 모든 예외를 무시합니다.

Added in version 3.12.

typedef int (*PyRefTracer)(PyObject*, int event, void *data)¶

:c:func:`PyRefTracer_SetTracer`를 사용하여 등록된 추적 함수의 유형. 첫 번째 매개 변수는 방금 생성되었거나 (event**가 :c:data:`PyRefTracer_CREATE`로 설정된 경우) 또는 곧 파괴될 예정인 파이썬 객체입니다 (**event**가 :c:data:`PyRefTracer_DESTROY`로 설정된 경우). **data 인수는 :c:func:`PyRefTracer_SetTracer`가 호출되었을 때 제공된 불투명 포인터입니다.

새로운 추적 함수가 현재 함수를 대체하여 등록되는 경우, 객체(object)가 NULL 로 설정되고 event 가 PyRefTracer_TRACKER_REMOVED 로 설정된 호출이 수행됩니다. 이는 새로운 함수가 등록되기 직전에 발생합니다.

Added in version 3.13.

int PyRefTracer_CREATE¶

Python 객체가 생성되었을 때 PyRefTracer 함수에서 event 매개변수의 값입니다.

int PyRefTracer_DESTROY¶

Python 객체가 파괴되었을 때 PyRefTracer 함수에서 event 매개변수의 값입니다.

int PyRefTracer_TRACKER_REMOVED¶

현재 트레이서가 새 트레이서로 대체되려고 할 때 PyRefTracer 함수에서 event 매개변수의 값입니다.

Added in version 3.14.

int PyRefTracer_SetTracer(PyRefTracer tracer, void *data)¶

참조 트레이서 함수를 등록합니다. 이 함수는 새로운 Python 객체가 생성되거나 객체가 파괴되려고 할 때 호출됩니다. data 가 제공되는 경우, 이는 트레이서 함수가 호출될 때 제공되는 불투명한 포인터여야 합니다. 성공하면 0 을 반환합니다. 예외가 발생하면 예외를 설정하고 -1 을 반환합니다.

Note that tracer functions must not create Python objects inside or otherwise the call will be re-entrant. The tracer also must not clear any existing exception or set an exception. A thread state will be active every time the tracer function is called.

There must be an attached thread state when calling this function.

이전에 다른 트레이서 함수가 등록된 경우, 새 함수가 등록되기 직전에 이전 함수가 event 가 PyRefTracer_TRACKER_REMOVED 로 설정된 호출을 받게 됩니다.

Added in version 3.13.

PyRefTracer PyRefTracer_GetTracer(void **data)¶

:c:func:`PyRefTracer_SetTracer`가 호출되었을 때 등록된 참조 트레이서 함수와 불투명 데이터 포인터의 값을 얻습니다. 트레이서가 등록되지 않은 경우 이 함수는 NULL을 반환하고 data 포인터를 NULL로 설정합니다.

There must be an attached thread state when calling this function.

Added in version 3.13.
