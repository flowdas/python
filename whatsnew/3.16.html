Python 3.16의 새로운 기능에 대해 알아봅시다¶

편집기:

추후 결정

이 문서는 파이썬 3.15와 비교한 파이썬 3.16의 새로운 기능들을 설명합니다.

자세한 내용은 :ref:`changelog <changelog>`를 참조하십시오.

참고

프리릴리스 사용자는 이 문서가 현재 초안 형식임을 인지해야 하며, Python 3.16 출시로 향함에 따라 상당한 업데이트가 있을 예정이므로 이전 버전을 읽은 후에도 다시 확인해 볼 가치가 있습니다.

기타 언어 변경 사항¶

새 모듈¶

  • 아직 없음.

개선된 모듈¶

gzip¶

  • gzip.open`은 이제 선택적 인자 ``mtime``을 받으며, 이는 :class:`~gzip.GzipFile() 클래스의 생성자에 전달됩니다. (Marin Misur 작성, gh-91372).

lzma¶

  • lzma.FILTER_ARM64 와 lzma.FILTER_RISCV 를 통해 새로운 BCJ 필터 ARM64 및 RISC-V를 지원합니다. 새 필터는 런타임 라이브러리가 이를 지원하는 경우에만 작동한다는 점에 유의하세요. ARM64 필터는 lzma 5.4.0 이상을 필요로 하는 반면, RISC-V는 5.6.0 이상을 필요로 합니다. (Chien Wong이 gh-115988 에서 기여했습니다.)

로깅¶

  • :class:`~logging.handlers.TimedRotatingFileHandler``는 이제 핸들러 생성 후 첫 번째 회전의 기준을 기존 로그 파일의 마지막 수정 시간 대신 생성 시간을 사용합니다. 이렇게 함으로써, 회전 간격이 만료되기 전에 시작하고 끝나는 짧게 실행되는 프로그램에서도 사용할 수 있게 합니다. (Iván Márton과 Serhiy Storchaka가 :gh:`84649`에서 기여했습니다.)

수학¶

  • 라디안 단위가 아닌 반 사이클 단위로 작동하는 삼각 함수를 추가했습니다. 새로운 함수인 math.acospi(), math.asinpi(), math.atanpi(), 그리고 math.atan2pi() 는 반 사이클 각도를 반환합니다. 새로운 함수인 math.cospi(), math.sinpi(), 및 math.tanpi() 는 반 사이클 각도 인수를 받습니다. 이 함수들은 IEEE 754-2019에서 권장되고 C23에서 표준화되었습니다. (Jeff Epler가 gh-150534 에서 기여했습니다.)

os¶

  • pidfd를 통해 다른 프로세스로부터 파일 기술자(file descriptor)를 복제하기 위한 :func:`os.pidfd_getfd``를 추가했습니다. Linux 5.6 이상에서 사용 가능합니다. (Maurycy Pawłowski-Wieroński가 :gh:`149464`에서 기여했습니다.)

shlex¶

  • 따옴표 없이도 셸에 안전한 문자열이라 하더라도 강제로 따옴표를 사용하도록 shlex.quote() 에 키워드 전용 매개변수 force 를 추가했습니다. (Jay Berry가 gh-148846 에서 기여했습니다.)

xml¶

  • Add support for multiple multi-byte encodings in the XML parser: “cp932”, “cp949”, “cp950”, “Big5”,”EUC-JP”, “GB2312”, “GBK”, “johab”, and “Shift_JIS”. Add partial support (only BMP characters) for multi-byte encodings “Big5-HKSCS”, “EUC_JIS-2004”, “EUC_JISX0213”, “Shift_JIS-2004”, “Shift_JISX0213”, “utf-8-sig” and non-standard aliases like “UTF8” (without hyphen). The parser now raises ValueError for known unsupported multi-byte encodings such us “ISO-2022-JP” or “raw-unicode-escape” instead of failing later, when encounter non-ASCII data. (Contributed by Serhiy Storchaka in gh-62259.)

제거됨¶

annotationlib¶

  • :meth`:meth:!annotationlib.ForwardRef._evaluate 메서드는 파이썬 3.14부터 사용 지원 중단(deprecated)되었습니다. 대신 :meth`:meth:annotationlib.ForwardRef.evaluate 또는 :func`:func:typing.evaluate_forward_ref 를 사용하시기 바랍니다.

배열¶

  • 문서에서는 파이썬 3.3부터, 런타임에서는 파이썬 3.13부터 사용 지원 중단된 'u' 포맷 코드 (wchar_t)를 사용하지 마십시오. 대신 'w' 포맷 코드 (Py_UCS4, 항상 4바이트)를 사용하십시오.

asyncio¶

  • :func`:func:!asyncio.iscoroutinefunction 는 파이썬 3.14부터 사용 지원 중단(deprecated)되었습니다. 대신 :func`:func:inspect.iscoroutinefunction 를 사용하십시오.

functools¶

  • function 또는 sequence 를 키워드 인자로 :func`:func:functools.reduce 의 파이썬 구현을 호출하는 것은 파이썬 3.14부터 사용 지원 중단되었습니다.

로깅¶

  • strm 인자를 이용한 사용자 정의 로깅 처리기 지원은 사용 지원 중단되었으며 파이썬 3.16에서 제거될 예정입니다. 대신 stream 인자를 사용하여 처리기를 정의하십시오.

mimetypes¶

  • :meth`:meth:mimetypes.MimeTypes.add_type 의 유효한 확장자는 . \으로 시작하거나 빈 문자열이어야 합니다. 점으로 시작하지 않는 확장자는 이제 ValueError 를 발생시킵니다.

shutil¶

  • :exc`:exc:!ExecError 예외는 파이썬 3.14부터 사용 지원 중단(deprecated)되었습니다. :mod`:mod:!shutil 의 어떤 함수에서도 파이썬 3.4 이후로 사용되지 않았습니다. (Stan Ulbrych가 gh-149567 \을 통해 기능을 구현하였습니다.)

symtable¶

  • :meth`:meth:!symtable.Class.get_methods 메서드는 파이썬 3.14부터 사용 지원 중단(deprecated)되었습니다.

sys¶

  • :func`:func:!_enablelegacywindowsfsencoding 함수는 파이썬 3.13부터 사용 지원 중단(deprecated)되었습니다. 대신 :envvar`:envvar:PYTHONLEGACYWINDOWSFSENCODING 환경 변수를 사용하십시오. (Stan Ulbrych가 gh-149595 \을 통해 기능을 구현하였습니다.)

sysconfig¶

  • :func`:func:!sysconfig.expand_makefile_vars 함수는 파이썬 3.14부터 사용 지원 중단(deprecated)되었습니다. 대신 :func`:func:sysconfig.get_paths 의 vars 인자를 사용하십시오. (Stan Ulbrych가 gh-149499 \을 통해 기능을 구현하였습니다.)

tarfile¶

  • 문서화되지도 사용되지도 않은 :attr`:attr:!tarfile.TarFile.tarfile 어트리뷰트는 파이썬 3.13부터 사용 지원 중단(deprecated)되었습니다.

폐지됨¶

  • abc

    • Python 3.3 이후부터 소프트 폐지된 :class`:class:abc.abstractclassmethod, :class`:class:abc.abstractstaticmethod, 및 :class`:class:abc.abstractproperty 는 이제 DeprecationWarning 를 발생시킵니다. 이 클래스들은 파이썬 3.21에서 제거될 예정이며, 대신 각각 :func`:func:abc.abstractmethod 와 함께 :func`:func:classmethod, :func`:func:staticmethod,および :class`:class:property 를 사용하십시오.

  • ast:

    • 파이썬 3.9부터 폐지된 클래스인 slice, Index, ExtSlice, Suite, Param, AugLoad 및 `AugStore 는 더 이상 from ast import * 에 의해 가져와지지 않으며 사용 시 폐기 경고를 발생시킵니다. 이 클래스들은 파이썬 3.21에 제거될 예정입니다. 이러한 타입은 파서에 의해 생성되지 않았거나 코드 생성기에 의해 받아들여지지 않습니다.

    • 파이썬 3.9부터 폐지된 ast.Tuple 객체의 dims 속성은 사용 시 폐기 경고를 발생시킵니다. 이 속성은 3.21에 제거될 예정입니다. 대신 ast.Tuple.elts 를 사용하십시오.

파이썬 3.17에서 제거 예정인 항목¶

  • :mod`:mod:datetime:

    • 연도를 포함하지 않는 날짜(‘일’)을 포함하는 형식 문자열로 :meth`:meth:~datetime.datetime.strptime 를 호출한 것은 파이썬 3.15부터 사용 지원 중단되었습니다. (Stan Ulbrych가 gh-70647 \을 통해 기능을 구현하였습니다.)

  • :mod`:mod:collections.abc:

    • :class`:class:collections.abc.ByteString 은 파이썬 3.17에서 제거될 예정입니다.

      isinstance(obj, collections.abc.Buffer)\를 사용하여 obj`가 실행 시점 :ref:`buffer 프로토콜 <bufferobjects>`를 구현하는지 테스트합니다. 타입 어노테이션을 사용하려면, :class:`~collections.abc.Buffer`를 사용하거나 코드에서 지원하는 타입을 명시적으로 지정하는 유니온 (예: ``bytes | bytearray | memoryview`)을 사용하십시오.

      ByteString`은 원래 :class:`bytes 및 bytearray 모두의 슈퍼타입 역할을 할 추상 클래스로 의도되었습니다. 그러나 ABC에 메서드가 전혀 없었기 때문에 객체가 ByteString`의 인스턴스라는 것을 알아내는 것이 해당 객체에 대해 유용한 정보를 제공한 적이 없습니다. 또한, :class:`memoryview`와 같은 다른 일반적인 버퍼 타입들 역시 (실행 시점 또는 정적 타입 검사기에 의해) :class:!ByteString`의 하위 타입으로 이해된 적이 없었습니다.

      자세한 내용은 :pep:`PEP 688 <688#current-options>`을 참조하십시오. (Shantanu Jain가 :gh:`91896`\를 통해 기능을 구현하였습니다.)

  • encodings:

    • 비-ASCII encoding 이름들을 encodings.normalize_encoding`에 전달하는 것은 더 이상 권장되지 않으며 파이썬 3.17에서 제거될 예정입니다. (Stan Ulbrych가 :gh:`136702()\를 통해 기능을 구현하였습니다.)

  • webbrowser:

    • webbrowser.MacOSXOSAScript`는 대신 :class:!webbrowser.MacOS`를 사용하도록 폐지되었습니다. (gh-137586)

  • typing:

    • 파이썬 3.14 이전에는 이전 스타일의 유니언이 비공개 클래스 typing._UnionGenericAlias\를 사용하여 구현되었습니다. 해당 클래스는 구현에 더 이상 필요하지 않지만, 하위 호환성을 위해 유지되어 왔으며 파이썬 3.17에서 제거될 예정입니다. 사용자는 비공개 구현 세부 사항에 의존하기보다는 typing.get_origin() 및 :func:`typing.get_args`와 같은 문서화된 인트로스펙션 도우미를 사용해 주시기를 부탁드립니다.

    • :class:`typing.ByteString`은 파이썬 3.9부터 폐지되었으며, 파이썬 3.17에서 제거될 예정입니다.

      isinstance(obj, collections.abc.Buffer)\를 사용하여 obj`가 실행 시점 :ref:`buffer 프로토콜 <bufferobjects>`를 구현하는지 테스트합니다. 타입 어노테이션을 사용하려면, :class:`~collections.abc.Buffer`를 사용하거나 코드에서 지원하는 타입을 명시적으로 지정하는 유니온 (예: ``bytes | bytearray | memoryview`)을 사용하십시오.

      ByteString`은 원래 :class:`bytes 및 bytearray 모두의 슈퍼타입 역할을 할 추상 클래스로 의도되었습니다. 그러나 ABC에 메서드가 전혀 없었기 때문에 객체가 ByteString`의 인스턴스라는 것을 알아내는 것이 해당 객체에 대해 유용한 정보를 제공한 적이 없습니다. 또한, :class:`memoryview`와 같은 다른 일반적인 버퍼 타입들 역시 (실행 시점 또는 정적 타입 검사기에 의해) :class:!ByteString`의 하위 타입으로 이해된 적이 없었습니다.

      자세한 내용은 :pep:`PEP 688 <688#current-options>`을 참조하십시오. (Shantanu Jain가 :gh:`91896`\를 통해 기능을 구현하였습니다.)

파이썬 3.18에서 제거 예정인 항목들¶

  • 더 이상 파일 디스크립터가 예상되는 곳에 불리언 값을 허용하지 않습니다. (Serhiy Storchaka가 gh-82626\를 통해 기능을 구현하였습니다.)

  • decimal:

    • 비표준적이고 문서화되지 않은 Decimal 형식 지정자 ``’N’``은 파이썬 3.13부터 폐지되었습니다. 이 지정자는 decimal 모듈의 C 구현에서만 지원됩니다. (Serhiy Storchaka가 gh-89902\를 통해 기능을 구현하였습니다.)

  • :pep:`829`에 의해 정의된 폐지 사항들:

    • name.pth 파일의 import 라인은 조용히 무시됩니다.

    (Barry Warsaw가 gh-148641\를 통해 기능을 구현하였습니다.)

파이썬 3.19에서 제거 예정인 항목들¶

  • ctypes:

    • 비-Windows 플랫폼에서 _pack_\를 설정하지만 _layout_\는 설정하지 않아 발생하는 암묵적인 MSVC 호환 구조 레이아웃 전환이 이제 사용되지 않으며, 파이썬 3.19에 제거될 예정입니다.

  • hashlib:

    • new() 또는 md5() 및 sha256`와 같은 직접 해시 명명된 생성자와 같은 해시 함수 생성자에서, 선택적인 초기 데이터 매개 변수에는 다양한 :mod:()!hashlib` 구현에서 data= 또는 ``string=``이라는 키워드 인자를 전달할 수도 있습니다.

      string 키워드 인자 이름에 대한 지원은 이제 더 이상 권장되지 않으며 파이썬 3.19에서 제거될 예정입니다.

      파이썬 3.13 이전에는 해시 함수의 백엔드 구현에 따라 string 키워드 매개변수가 올바르게 지원되지 않았습니다. 최대의 하위 호환성을 위해 초기 데이터를 위치 인수(positional argument)로 전달하는 것을 선호하십시오.

  • http.cookies:

    • :meth:`http.cookies.Morsel.js_output`는 폐지되었으며 파이썬 3.19에서 제거될 예정입니다.

    • :meth:`http.cookies.BaseCookie.js_output`은 폐지되었으며 파이썬 3.19에서 제거될 예정입니다.

  • imaplib:

    • :attr:`IMAP4.file <imaplib.IMAP4.file>`을 변경하는 것은 이제 사용되지 않게 되었으며 파이썬 3.19에서 제거될 예정입니다. 이 속성은 더 이상 사용되지 않으며 값을 변경해도 현재 파일은 자동으로 닫히지 않습니다.

      파이썬 3.14 이전에는 이 속성이 IMAP4`에 해당하는 ``read()` 및 readline() 메서드를 구현하는 데 사용되었지만, 그 이후로는 더 이상 그렇지 않습니다.

파이썬 3.20에서 제거 예정인 항목들¶

향후 버전에서 제거 예정인 항목들¶

다음 API들은 향후 제거될 것이지만, 현재 제거 일정이 정해져 있지 않습니다.

  • argparse:

    • 인자 그룹 중첩 및 상호 배타적 그룹 중첩은 사용 지원 중단되었습니다.

    • :meth:`~argparse.ArgumentParser.add_argument_group`에 문서화되지 않은 키워드 인자 prefix_chars\를 전달하는 것은 이제 사용 지원 중단되었습니다.

    • argparse.FileType 타입 변환기가 사용 지원 중단되었습니다.

  • builtins:

    • 제너레이터: throw(type, exc, tb) 및 athrow(type, exc, tb) 시그니처는 사용 지원 중단되었습니다. 대신 단일 인자 시그니처인 throw(exc) 및 ``athrow(exc)``를 사용하십시오.

    • 현재 파이썬은 숫자 리터럴 뒤에 키워드를 즉시 허용합니다 (예: 0in x, 1or x, 0if 1else 2). 이는 [0x1for x in y]``와 같이 혼란스럽고 모호한 표현식도 허용합니다 (이는 ``[0x1 for x in y] 또는 ``[0x1f or x in y]``로 해석될 수 있습니다). 숫자 리터럴 뒤에 키워드 and, else, for, if, in, is, 및 or 중 하나가 즉시 따라오는 경우 구문 경고가 발생합니다. 향후 릴리스에서는 문법 오류로 변경될 것입니다. (gh-87999)

    • 비-정수 타입을 반환하는 __index__() 및 __int__() 메서드에 대한 지원: 이 메서드는 :class:`int`의 엄격한 하위 클래스(strict subclass) 인스턴스를 반환하도록 요구될 것입니다.

    • float`의 엄격한 하위 클래스를 반환하는 ``__float__()` 메서드에 대한 지원: 이 메서드는 float 인스턴스를 반환하도록 요구될 것입니다.

    • complex`의 엄격한 하위 클래스를 반환하는 ``__complex__()` 메서드에 대한 지원: 이 메서드는 complex 인스턴스를 반환하도록 요구될 것입니다.

    • 복소수를 complex() 생성자의 real 또는 imag 인자로 전달하는 것은 이제 사용 지원 중단되었습니다. 단일 위치 인자로서만 전달해야 합니다. (Serhiy Storchaka가 gh-109218\를 통해 기능을 구현하였습니다.)

  • calendar: calendar.January 및 calendar.February 상수는 사용 지원 중단되었으며, calendar.JANUARY 및 calendar.FEBRUARY`로 대체되었습니다. (Prince Roshan가 :gh:`103636\를 통해 기능을 구현하였습니다.)

  • codecs: 대신 open`을 사용하십시오. (:gh:`133038())

Python 3.16으로의 이식¶

이 섹션은 이전에 설명된 변경 사항 및 코드에 변경이 필요할 수 있는 기타 버그 수정 목록을 제공합니다.

빌드 변경 사항¶

  • 유지보수와 업데이트를 단순화하기 위해 CPython 소스 트리의 번들된 libmpdec decimal 라이브러리 복사본을 제거합니다. decimal 모듈은 이제 시스템의 libmpdec decimal 라이브러리를 무조건 사용하게 됩니다. 또한 더 이상 사용되지 않는 --with-system-libmpdec configure 플래그도 제거됩니다. 이 변경 사항은 과거 몇 번의 릴리스 동안 별도의 libmpdec 사본을 기반으로 빌드된 파이썬의 바이너리 릴리스에는 영향을 미치지 않습니다.

    (Sergey B Kirpichev가 gh-115119\를 통해 기능을 구현하였습니다.)

  • 동일한 트리에 파이썬의 여러 버전을 공동 설치하는 리눅스 배포판들이 build-details.json 충돌을 방지할 수 있도록 --with-build-details-suffix configure 플래그를 추가합니다.

    (Stefano Rivera가 gh-131372\를 통해 기능을 구현하였습니다.)
