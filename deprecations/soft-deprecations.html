Soft deprecations (약하게 폐지된)¶

soft deprecated API를 제거할 계획은 없습니다.

  • re.match() 및 re.Pattern.match() 는 이제 더 명시적인 대체 이름으로 추가된 새로운 re.prefixmatch() 및 re.Pattern.prefixmatch() API를 선호하며 soft deprecated 상태입니다. 이는 Python의 “명시적인 것이 암시적이다보다 낫다”라는 정신을 따라 match 가 무엇을 의미하는지 혼란을 줄이기 위해 사용됩니다. 대부분의 다른 언어 정규 표현식 라이브러리는 Python이 항상 search 라고 불렀던 것을 의미하기 위해 match 라는 이름을 가진 API를 사용합니다.

    우리는 30년 이상 코드에서 사용되어 왔기 때문에, 구형 match() 이름을 제거할 계획이 없습니다. 구형 Python 버전을 지원하는 코드는 계속 match`를 사용해야 하며, 새로운 코드는 :func:()!prefixmatch`를 선호해야 합니다. :ref:`prefixmatch-vs-match`를 참조하십시오.

    (Gregory P. Smith가 :gh:`86519`를 통해 기여하고 Hugo van Kemenade가 :gh:`148100`를 통해 기여했습니다.)
