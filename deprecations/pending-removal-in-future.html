향후 버전에서 제거 예정인 항목들¶

다음 API들은 향후 제거될 것이지만, 현재 제거 일정이 정해져 있지 않습니다.

  • argparse:

    • 인자 그룹 중첩 및 상호 배타적 그룹 중첩은 사용 지원 중단되었습니다.

    • :meth:`~argparse.ArgumentParser.add_argument_group`에 문서화되지 않은 키워드 인자 prefix_chars\를 전달하는 것은 이제 사용 지원 중단되었습니다.

    • argparse.FileType 타입 변환기가 사용 지원 중단되었습니다.

  • builtins:

    • 제너레이터: throw(type, exc, tb) 및 athrow(type, exc, tb) 시그니처는 사용 지원 중단되었습니다. 대신 단일 인자 시그니처인 throw(exc) 및 ``athrow(exc)``를 사용하십시오.

    • 현재 파이썬은 숫자 리터럴 뒤에 키워드를 즉시 허용합니다 (예: 0in x, 1or x, 0if 1else 2). 이는 [0x1for x in y]``와 같이 혼란스럽고 모호한 표현식도 허용합니다 (이는 ``[0x1 for x in y] 또는 ``[0x1f or x in y]``로 해석될 수 있습니다). 숫자 리터럴 뒤에 키워드 and, else, for, if, in, is, 및 or 중 하나가 즉시 따라오는 경우 구문 경고가 발생합니다. 향후 릴리스에서는 문법 오류로 변경될 것입니다. (gh-87999)

    • 비-정수 타입을 반환하는 __index__() 및 __int__() 메서드에 대한 지원: 이 메서드는 :class:`int`의 엄격한 하위 클래스(strict subclass) 인스턴스를 반환하도록 요구될 것입니다.

    • float`의 엄격한 하위 클래스를 반환하는 ``__float__()` 메서드에 대한 지원: 이 메서드는 float 인스턴스를 반환하도록 요구될 것입니다.

    • complex`의 엄격한 하위 클래스를 반환하는 ``__complex__()` 메서드에 대한 지원: 이 메서드는 complex 인스턴스를 반환하도록 요구될 것입니다.

    • 복소수를 complex() 생성자의 real 또는 imag 인자로 전달하는 것은 이제 사용 지원 중단되었습니다. 단일 위치 인자로서만 전달해야 합니다. (Serhiy Storchaka가 gh-109218\를 통해 기능을 구현하였습니다.)

  • calendar: calendar.January 및 calendar.February 상수는 사용 지원 중단되었으며, calendar.JANUARY 및 calendar.FEBRUARY`로 대체되었습니다. (Prince Roshan가 :gh:`103636\를 통해 기능을 구현하였습니다.)

  • codecs: use open() instead of codecs.open(). (gh-133038)

  • codeobject.co_lnotab: 대신 codeobject.co_lines() 메서드를 사용해주세요.

  • :mod`:mod:datetime:

    • utcnow(): datetime.datetime.now(tz=datetime.UTC):를 사용하십시오.

    • utcfromtimestamp(): datetime.datetime.fromtimestamp(timestamp, tz=datetime.UTC):를 사용하십시오.

  • gettext: 복수형 값은 정수여야 합니다.

  • importlib:

    • :func:`~importlib.util.cache_from_source`의 debug_override 매개변수는 사용 지원 중단되었습니다. 대신 optimization 매개변수를 사용하십시오.

  • importlib.metadata:

    • EntryPoints 튜플 인터페이스.

    • return 값의 암묵적인 None.

  • logging: warn() 메서드는 Python 3.3부터 사용 지원 중단되어, 대신 :meth:`~logging.warning`을 사용하십시오.

  • mailbox: StringIO 입력 및 텍스트 모드 사용은 사용 지원 중단되었으며, 대신 BytesIO와 바이너리 모드를 사용하십시오.

  • os: 다중 스레드 프로세스에서 os.register_at_fork()\를 호출하는 것은 지원이 중단되었습니다 (deprecated).

  • os.path: os.path.commonprefix`는 이제 사용이 중단되었으며, 경로 접두사에는 :func:`os.path.commonpath`를 사용해 주십시오. :func:`os.path.commonprefix() 함수는 이름과 모듈이 잘못되어 사용이 중단되고 있습니다. 이 함수는 경로 조작에 관한 모듈에 포함되어 있지만, 공통 접두사를 반환하기 때문에 사용 시 실수로 지나친 디렉터리 탐색 취약점을 파이썬 프로그램에 도입하기 쉽습니다.

  • pydoc.ErrorDuringImport: exc_info 매개변수에 대한 튜플 값은 더 이상 사용되지 않으며, 예외 인스턴스를 사용해 주십시오.

  • re: 정규 표현식에서 숫자 그룹 참조 및 그룹 이름에 대해 더욱 엄격한 규칙이 적용됩니다. 이제 ASCII 숫자의 순차적인 시퀀스만 숫자 참조로 받아들여집니다. 바이트 패턴 및 대체 문자열의 그룹 이름은 이제 ASCII 알파벳과 숫자, 언더바만을 포함할 수 있습니다. (Serhiy Storchaka가 gh-91760\를 통해 기능을 구현하였습니다.)

  • shutil: :func:`~shutil.rmtree`의 onerror 매개변수는 Python 3.12에서 더 이상 사용되지 않습니다 (deprecated). 대신 onexc 매개변수를 사용해 주십시오.

  • ssl 옵션 및 프로토콜:

    • 프로토콜 인자 없이 :class:`ssl.SSLContext`를 사용하는 것은 더 이상 사용되지 않습니다 (deprecated).

    • ssl.SSLContext: set_npn_protocols() 및 :meth:`!selected_npn_protocol`은 더 이상 사용되지 않습니다 (deprecated). 대신 ALPN을 사용해 주십시오.

    • ssl.OP_NO_SSL* 옵션

    • ssl.OP_NO_TLS* 옵션

    • ssl.PROTOCOL_SSLv3

    • ssl.PROTOCOL_TLS

    • ssl.PROTOCOL_TLSv1

    • ssl.PROTOCOL_TLSv1_1

    • ssl.PROTOCOL_TLSv1_2

    • ssl.TLSVersion.SSLv3

    • ssl.TLSVersion.TLSv1

    • ssl.TLSVersion.TLSv1_1

  • threading 메서드:

    • threading.Condition.notifyAll(): 대신 notify_all()\를 사용해 주십시오.

    • threading.Event.isSet(): 대신 is_set()\를 사용해 주십시오.

    • threading.Thread.isDaemon(), threading.Thread.setDaemon(): 대신 threading.Thread.daemon 속성을 사용하십시오.

    • threading.Thread.getName(), threading.Thread.setName(): 대신 threading.Thread.name 속성을 사용하십시오.

    • threading.currentThread(): 대신 threading.current_thread()\를 사용해 주십시오.

    • threading.activeCount(): 대신 threading.active_count()\를 사용해 주십시오.

  • typing.Text (gh-92332).

  • 내부 클래스 ``typing._UnionGenericAlias``는 더 이상 typing.Union`을 구현하는 데 사용되지 않습니다. 이 비공개 클래스를 사용하는 사용자들과의 호환성을 유지하기 위해, 최소 Python 3.17까지 호환성 쉬м(shim)이 제공될 것입니다. (Jelle Zijlstra가 :gh:`105499\를 통해 기능을 구현하였습니다.)

  • unittest.IsolatedAsyncioTestCase: 테스트 케이스에서 ``None``이 아닌 값을 반환하는 것은 더 이상 사용되지 않습니다 (deprecated).

  • :mod:`urllib.parse`의 사용 중단된 함수들: 대신 :func:`~urllib.parse.urlparse`를 사용하십시오.

    • splitattr()

    • splithost()

    • splitnport()

    • splitpasswd()

    • splitport()

    • splitquery()

    • splittag()

    • splittype()

    • splituser()

    • splitvalue()

    • to_bytes()

  • wsgiref: ``SimpleHandler.stdout.write()``는 부분 쓰기를 수행해서는 안 됩니다.

  • xml.etree.ElementTree: 대신 명시적인 ``len(elem)` 또는 elem is not None 테스트를 선호해 주십시오.

  • :func:`sys._clear_type_cache`는 더 이상 사용되지 않습니다 (deprecated): 대신 :func:`sys._clear_internal_caches`를 사용해 주십시오.
