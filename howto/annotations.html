어노테이션 모범 사례¶

추상

이 문서는 주석 딕셔너리를 다루는 모범 사례를 캡슐화하도록 설계되었습니다. 파이썬 객체에서 __annotations__ 를 검사하는 파이썬 코드를 작성하는 경우, 아래에 설명된 가이드라인을 따르기를 권장합니다.

이 문서는 네 가지 섹션으로 구성되어 있습니다: Python 3.10 이상 버전에서 객체의 어노테이션에 접근하기 위한 모범 사례, Python 3.9 이하 버전에서 객체의 어노테이션에 접근하기 위한 모범 사례, 모든 Python 버전에 적용되는 __annotations__ 에 대한 다른 모범 사례, 그리고 __annotations__ 의 특이 사항입니다.

이 문서는 __annotations__ 를 다루는 것에만 국한됨을 유념하세요. 어노테이션 용 사용에 관한 것은 아닙니다. 코드에서 “타입 힌트”를 사용하는 방법에 대한 정보가 필요하면 typing 모듈을 참조하세요.

Python 3.10 이상에서 객체의 어노테이션 딕셔너리에 접근하기¶

Python 3.10은 표준 라이브러리에 새 함수인 :func:`inspect.get_annotations`를 추가했습니다. Python 3.10부터 3.13까지 버전에서는 이 함수를 호출하는 것이 어노테이션을 지원하는 모든 객체의 어노테이션 딕셔너리 접근을 위한 모범 사례입니다. 이 함수는 문자열 형식의 어노테이션을 “언-문자열화”할 수도 있습니다.

Python 3.14에서는 어노테이션을 다루는 기능을 갖춘 새로운 annotationlib 모듈이 있습니다. 여기에는 annotationlib.get_annotations() 함수가 포함되며, 이 함수는 :func:`inspect.get_annotations`를 대체합니다.

만약 어떤 이유로든 inspect.get_annotations`가 사용 사례에 적합하지 않다면, ``__annotations__`() 데이터 멤버에 수동으로 접근할 수 있습니다. 이것 역시 Python 3.10에서 모범 사례가 변경되었습니다. Python 3.10부터 `o.__annotations__`는 Python 함수, 클래스 및 모듈에서 항상 작동하는 것이 보장됩니다. 검사하는 객체가 이 세 가지 특정 객체 중 하나임을 확신한다면, 단순히 `o.__annotations__`를 사용하여 객체의 어노테이션 딕셔너리에 접근할 수 있습니다.

하지만, functools.partial() 에 의해 생성된 콜러블과 같은 다른 유형의 콜러블은 __annotations__ 속성을 가지고 있지 않을 수 있습니다. 알 수 없는 객체의 __annotations__ 에 접근할 때, Python 3.10 이상 버전의 모범 사례는 세 개의 인수를 사용하여 getattr() 를 호출하는 것입니다. 예를 들어, getattr(o, '__annotations__', None) 를 사용합니다.

Python 3.10 이전에는 어노테이션이 정의되지 않은 클래스에 대해, 하지만 부모 클래스에 어노테이션이 있는 경우, __annotations__ 에 접근하면 부모의 __annotations__ 가 반환되었습니다. Python 3.10 이상 버전에서는 자식 클래스의 어노테이션이 빈 딕셔너리(empty dict)일 것입니다.

Python 3.9 이하에서 객체의 어노테이션 딕셔너리에 접근하기¶

파이썬 3.9 이전에서는 객체의 어노테이션 딕셔너리에 액세스하는 것이 신 버전보다 훨씬 복잡합니다. 이 문제는 클래스 어노테이션과 특히 관련된, 이러한 이전 버전의 파이썬의 디자인 결함입니다.

다른 객체들(함수, 다른 호출 가능한 객체, 모듈 등)의 어노테이션 딕셔너리에 액세스하는 모범 사례는 inspect.get_annotations`를 호출하지 않는다고 가정할 때 3.10과 동일합니다. 객체의 ``__annotations__`() 애트리뷰스에 액세스하기 위해 세 인자짜리 :func:`getattr`를 사용해야 합니다.

유감스럽게도, 이것은 클래스에서는 모범 사례가 아닙니다. 문제는 __annotations__ 가 클래스에서 선택 사항이고, 클래스가 기본 클래스로부터 애트리뷰트를 상속받을 수 있기 때문에, 클래스의 __annotations__ 애트리뷰스에 액세스하는 것이 실수로 기본 클래스 의 어노테이션 딕셔너리를 반환할 수 있다는 점입니다. 예를 들면

class Base:
    a: int = 3
    b: str = 'abc'

class Derived(Base):
    pass

print(Derived.__annotations__)

이것은 Base 의 어노테이션 딕셔너리를 출력하고, Derived 의 것은 아닙니다.

코드에 명시하는 객체가 클래스인지(isinstance(o, type))에 따라 별도의 코드 경로가 필요합니다. 이 경우, 모범 사례는 파이썬 3.9 이전의 구현 세부 사항에 의존합니다. 즉, 클래스가 어노테이션이 정의되어 있다면, 그들은 클래스의 __dict__ 딕셔너리에 저장됩니다. 클래스는 어노테이션이 정의되어 있을 수도 있고 아닐 수도 있으므로, 모범 사례는 클래스 딕셔너리에서 get() 메서드를 호출하는 것입니다.

모든 것을 종합해서, 파이썬 3.9 이전에서 임의의 객체의 __annotations__ 애트리뷰스에 안전하게 액세스하는 샘플 코드가 여기 있습니다.

if isinstance(o, type):
    ann = o.__dict__.get('__annotations__', None)
else:
    ann = getattr(o, '__annotations__', None)

이 코드를 실행한 후, ann 은 딕셔너리이거나 None 일 것입니다. 추가 검토 전에 isinstance() 를 사용하여 ann 의 타입을 더블 체크하는 것이 좋습니다.

일부 특이하거나 malformed (잘못 구성된) 타입 객체는 __dict__ 애트리뷰스가 없을 수 있으므로, 추가 안전을 위해 getattr`를 사용하여 :attr:()!__dict__`에 액세스하는 것도 고려해 볼 수 있습니다.

문자열화된 어노테이션을 수동으로 역문자열화하기¶

일부 어노테이션이 “문자열화”된 상태일 수 있고, 해당 문자열을 평가하여 나타내는 파이썬 값을 얻고 싶다면, :func:`inspect.get_annotations`를 호출하여 이 작업을 수행하는 것이 가장 좋습니다.

만약 파이썬 3.9 이하 버전을 사용하거나, 어떤 이유로 인해 inspect.get_annotations`를 사용할 수 없다면, 그 로직을 복제해야 합니다. 현재 파이썬 버전의 :func:`inspect.get_annotations() 구현을 검토하고 유사한 접근 방식을 따르는 것이 좋습니다.

간단히 말해, 임의의 객체 o 에 대한 문자열화된 어노테이션을 평가하고 싶다면 다음과 같습니다.

  • o 가 모듈인 경우, eval() 을 호출할 때 globals 로 o.__dict__ 를 사용해야 합니다.

  • o 가 클래스인 경우, eval() 을 호출할 때 globals 로 sys.modules[o.__module__].__dict__ 를, locals 로 dict(vars(o)) 를 사용해야 합니다.

  • functools.update_wrapper(), @functools.wraps, 또는 functools.partial() 를 사용하는 래핑된 호출 가능한 객체인 경우, o.__wrapped__ 또는 적절한 o.func 에 액세스하여 반복적으로 언래핑하여, 루트 언래핑된 함수를 찾을 때까지 이를 수행해야 합니다.

  • o 가 호출 가능한 객체(하지만 클래스는 아닌 경우)인 경우, eval() 을 호출할 때 globals 로 o.__globals__ 를 사용합니다.

하지만, 어노테이션으로 사용되는 모든 문자열 값이 :func:`eval`에 의해 성공적으로 파이썬 값으로 변환될 수 있는 것은 아닙니다. 문자열 값은 이론적으로 모든 유효한 문자열을 포함할 수 있으며, 실제로는 문자열 값으로 어노테이션해야만 하는 타입 힌트의 유효한 사용 사례가 있습니다. 예를 들면:

  • 파이썬 3.10에 이 지원이 추가되기 전의 | 를 사용한 PEP 604 유니온 타입입니다.

  • 실시간에는 필요하지 않고, :const:`typing.TYPE_CHECKING`이 True일 때만 임포트되는 정의입니다.

만약 :func:`eval`이 이러한 값을 평가하려고 시도하면, 실패하고 예외를 발생시킵니다. 따라서 어노테이션을 다루는 라이브러리 API를 설계할 때는 호출자가 명시적으로 요청할 때만 문자열 값을 평가하려고 시도하는 것이 좋습니다.

모든 파이썬 버전에서 __annotations__ 에 대한 모범 사례¶

  • 객체에 __annotations__ 멤버를 직접 할당하는 것은 피해야 합니다. 대신 Python이 __annotations__ 설정을 관리하도록 맡겨주십시오.

  • 객체의 __annotations__ 멤버에 직접 할당하는 경우, 항상 dict 객체로 설정해야 합니다.

  • 어떤 객체에서든 __annotations__ 에 직접 접근하는 것을 피해야 합니다. 대신 annotationlib.get_annotations() (Python 3.14 이상) 또는 inspect.get_annotations() (Python 3.10 이상)을 사용하십시오.

  • 만약 객체의 __annotations__ 멤버에 직접 접근할 경우, 내용을 검사하기 전에 이 것이 딕셔너리인지 확인해야 합니다.

  • __annotations__ 딕셔너리를 수정하는 것은 피해야 합니다.

  • 객체의 __annotations__ 속성을 삭제하는 것은 피해야 합니다.

__annotations__ 의 특징들¶

Python 3의 모든 버전에서, 함수 객체는 해당 객체에 어노테이션이 정의되어 있지 않으면 lazy-create 방식으로 어노테이션 딕셔너리를 생성합니다. del fn.__annotations__ 를 사용하여 __annotations__ 속성을 삭제할 수 있지만, 이후에 fn.__annotations__ 에 접근하면 객체는 새로 생성된 빈 딕셔너리를 만들고 이를 어노테이션으로 저장하여 반환합니다. 함수에서 어노테이션을 lazy-create 하기 이전에 삭제하면 AttributeError 가 발생합니다. 연속으로 del fn.__annotations__ 를 두 번 사용하면 항상 AttributeError 가 발생합니다.

위 단락의 내용은 Python 3.10 및 그 이후 버전의 클래스와 모듈 객체에도 적용됩니다.

Python 3의 모든 버전에서, 함수 객체에 __annotations__ 를 None 으로 설정할 수 있습니다. 그러나 이후에 fn.__annotations__ 을 사용하여 해당 객체의 어노테이션에 접근하면 이 섹션의 첫 번째 단락에 따라 빈 딕셔너리가 lazy-create 됩니다. 이는 어떤 Python 버전에서도 모듈이나 클래스에는 적용되지 않습니다. 그러한 객체들은 __annotations__ 를 어떤 Python 값으로든 설정할 수 있으며, 설정된 값을 유지합니다.

Python이 어노테이션을 문자열화(from __future__ import annotations 사용)하고, 사용자가 어노테이션으로 문자열을 지정하는 경우, 이 문자열 자체가 따옴표로 감싸집니다. 즉, 어노테이션이 두 번 따옴표로 감싸지는 것입니다. 예를 들면:

from __future__ import annotations
def foo(a: "str"): pass

print(foo.__annotations__)

이것은 {'a': "'str'" } 를 출력합니다. 이것은 실제로 “특징”으로 간주되어서는 안 됩니다. 단지 놀라움을 줄 수 있기 때문에 여기에 언급된 것뿐입니다.

사용자 정의 메타클래스를 가진 클래스를 사용하고 클래스에서 __annotations__``에 접근하면 예상치 못한 동작을 관찰할 수 있습니다. 예시를 보려면 :pep:`749 <749#pep749-metaclasses>`를 참조하십시오. Python 3.14 이상에서는 :func:`annotationlib.get_annotations` 또는 Python 3.10 이상에서는 :func:`inspect.get_annotations`을 사용하여 이러한 특징들을 방지할 수 있습니다. 이전 버전의 Python에서는 클래스의 :attr:`~type.__dict__`에서 어노테이션에 접근하여 이러한 버그들을 피할 수 있습니다 (예를 들어, ``cls.__dict__.get('__annotations__', None)).

일부 버전의 Python에서는 클래스의 인스턴스가 __annotations__ 속성을 가질 수 있습니다. 하지만 이는 지원되는 기능이 아닙니다. 인스턴스의 어노테이션이 필요한 경우, type`을 사용하여 해당 클래스에 접근할 수 있습니다 (예를 들어, Python 3.14+에서 ``annotationlib.get_annotations(type(myinstance))`()).
