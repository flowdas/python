profiling — 파이썬 프로파일러¶

Added in version 3.15.

소스 코드: Lib/profiling/

프로파일링 소개¶

:dfn:`프로파일`은 프로그램의 여러 부분이 얼마나 자주 그리고 얼마나 오랫동안 실행되었는지를 기술하는 통계 집합입니다. 이러한 통계는 성능 병목 지점을 식별하고 최적화 노력을 안내하는 데 도움이 됩니다. 파이썬은 이 정보를 수집하기 위한 두 가지 근본적으로 다른 접근 방식을 제공합니다: 통계적 샘플링과 결정론적 추적.

profiling 패키지는 파이썬의 내장 프로파일링 도구를 단일 네임스페이스 아래에 정리합니다. 여기에는 각기 다른 프로파일링 방법론을 구현하는 두 개의 서브모듈이 포함됩니다:

profiling.sampling

호출 스택을 주기적으로 샘플링하는 통계적 프로파일러입니다. 스크립트를 직접 실행하거나 PID로 실행 중인 프로세스에 첨부할 수 있습니다. 거의 오버헤드 없이 여러 출력 형식(flame graphs, heatmaps, Firefox Profiler), GIL 분석, GC 추적 및 여러 프로파일링 모드(wall-clock, CPU, GIL)를 제공합니다.

profiling.tracing

모든 함수 호출, 반환 및 예외 이벤트를 추적하는 결정론적 프로파일러입니다. 정확한 호출 횟수와 정밀한 타이밍 정보를 제공하여, 매우 빠른 함수를 포함한 모든 호출을 포착합니다.

참고

프로파일러 모듈은 벤치마킹 목적이 아니라 주어진 프로그램에 대한 실행 프로파일을 제공하도록 설계되었습니다. 벤치마킹 시에는 합리적으로 정확한 타이밍 측정을 제공하는 timeit 모듈을 사용하세요. 이 구분은 파이썬 코드와 C 코드를 비교할 때 특히 중요합니다: 결정론적 프로파일러는 파이썬 코드에는 오버헤드를 도입하지만 C 수준 함수에는 도입하지 않으며, 이는 비교를 왜곡할 수 있습니다.

프로파일러 선택하기¶

대부분의 성능 분석에는 통계적 프로파일러 (profiling.sampling)를 사용하세요. 최소한의 오버헤드를 가지며 개발 및 운영 환경 모두에서 작동하고, flame graph, heatmaps, GIL 분석 등 풍부한 시각화 옵션을 제공합니다.

정확한 호출 횟수가 필요하고 함수 호출 중 하나라도 놓칠 여유가 없을 때는 결정론적 프로파일러 (profiling.tracing)를 사용하세요. 모든 함수 호출과 반환을 도구화하기 때문에, 샘플링 간격 사이에 완료되는 매우 빠른 함수까지 포착합니다. 단점은 오버헤드가 높다는 것입니다.

통계적 샘플링 사용 시점¶

성능 분석 작업의 대부분에는 통계적 프로파일러인 :mod:`profiling.sampling`을 권장합니다. 이는 :mod:`profiling.tracing`을 사용하는 방식과 동일하게 사용하십시오:

python -m profiling.sampling run script.py

샘플링 프로파일러의 주요 장점 중 하나는 다양한 출력 형식입니다. 전통적인 pstats 테이블 외에도, 호출 계층 구조를 시각화하는 인터랙티브 플레임 그래프, 코드에서 시간이 얼마나 사용되는지 정확하게 보여주는 라인 레벨 소스 히트맵, 타임라인 기반 분석을 위한 Firefox Profiler 출력을 생성할 수 있습니다.

또한 프로파일러는 결정론적 프로파일링이 포착할 수 없는 파이썬 인터프리터 동작 방식에 대한 통찰력을 제공합니다. --mode gil``을 사용하여 멀티스레드 코드의 GIL 경합을 식별하고, ``--mode cpu``를 사용하여 I/O 대기를 제외한 실제 CPU 시간을 측정하거나, ``<GC> 프레임을 검사하여 가비지 컬렉션 오버헤드를 이해할 수 있습니다. --native 옵션은 C 확장 프로그램에서 소요된 시간을 드러내어 파이썬 오버헤드와 라이브러리 성능을 구별하는 데 도움이 됩니다.

다중 스레드 애플리케이션의 경우, -a 옵션은 모든 스레드를 동시에 샘플링하여 작업이 어떻게 분산되는지 보여줍니다. 또한 프로덕션 디버깅을 위해 attach 명령은 재시작이나 코드 변경 없이 PID를 통해 실행 중인 모든 파이썬 프로세스에 연결합니다.

결정론적 추적을 사용해야 하는 경우¶

결정론적 프로파일러는 (profiling.tracing) 모든 함수 호출과 반환을 인스트루먼트합니다. 이 방식은 샘플링보다 오버헤드가 높지만, 프로그램 실행을 완전히 커버한다는 것을 보장합니다.

결정론적 추적을 선택하는 주된 이유는 정확한 호출 횟수가 필요할 때입니다. 통계적 프로파일링은 샘플링을 기반으로 빈도를 추정하므로, 샘플 사이에서 완료되는 수명이 짧은 함수를 과소 계산할 수 있습니다. 최적화가 실제로 함수 호출 횟수를 줄였는지 확인하거나, 호출자-피호출자 관계를 이해하기 위해 전체 호출 그래프를 추적하려면 결정론적 추적이 올바른 선택입니다.

결정론적 추적은 마이크로초 단위로 실행되는 함수를 포착하는 데도 탁월합니다. 이러한 함수는 통계적 샘플에서는 자주 나타나지 않을 수 있지만, 결정론적 추적은 지속 시간과 관계없이 모든 호출을 기록합니다.

빠른 시작¶

이 섹션은 프로파일링을 시작하는 데 필요한 최소 단계를 제공합니다. 전체 문서는 각 프로파일러 전용 페이지를 참조하십시오.

통계적 프로파일링¶

스크립트를 프로파일링하려면, run 명령을 사용하여 profiling.sampling 모듈을 사용하십시오:

python -m profiling.sampling run script.py
python -m profiling.sampling run -m mypackage.module

이것은 프로파일러 하에서 스크립트를 실행하고 시간이 어디에 소요되었는지에 대한 요약을 출력합니다. 인터랙티브 플레임 그래프를 생성하려면:

python -m profiling.sampling run --flamegraph script.py

이미 실행 중인 프로세스를 프로파일링하려면, 프로세스 ID와 함께 attach 명령을 사용하십시오:

python -m profiling.sampling attach 1234

사용자 지정 설정을 위해서는 샘플링 간격(마이크로초 단위)과 지속 시간(초 단위)을 지정하십시오:

python -m profiling.sampling run -i 50 -d 30 script.py

결정론적 프로파일링¶

명령줄에서 스크립트를 프로파일링하려면:

python -m profiling.tracing script.py

코드를 프로그램적으로 프로파일링하려면:

import profiling.tracing
profiling.tracing.run('my_function()')

이것은 주어진 코드를 프로파일러 하에서 실행하고 정확한 함수 호출 횟수와 타이밍을 보여주는 요약을 출력합니다.

프로파일 출력 이해하기¶

두 프로파일러 모두 함수 레벨 통계를 수집하지만, 표시하는 형식은 다릅니다. 샘플링 프로파일러는 여러 시각화(플레임 그래프, 히트맵, Firefox Profiler, pstats 테이블)를 제공하는 반면, 결정론적 프로파일러는 pstats와 호환되는 출력을 생성합니다. 형식과 관계없이 기본 개념은 동일합니다.

주요 프로파일링 개념:

직접 시간 (또는 self time 또는 tottime)

함수 자체에서 코드를 실행하는 데 소요된 시간이며, 해당 함수가 호출한 함수에서 소요된 시간은 제외됩니다. 높은 직접 시간은 해당 함수가 비용이 많이 드는 작업을 포함하고 있음을 나타냅니다.

누적 시간 (또는 total time 또는 cumtime)

함수와 해당 함수가 호출한 모든 함수에서 소요된 시간입니다. 이는 전체 호출 서브트리를 포함하여 함수를 호출하는 전체 비용을 측정합니다.

호출 횟수 (또는 ncalls 또는 samples)

함수가 호출된 횟수 (결정론적) 또는 샘플링된 횟수 (통계적)입니다. 결정론적 프로파일링의 경우, 이는 정확합니다. 통계적 프로파일링의 경우, 이는 스택 샘플에 함수가 나타난 횟수를 나타냅니다.

프리미티브 호출 수

재귀에 의해 유발되지 않은 호출 횟수입니다. 함수가 재귀할 경우, 총 호출 횟수에는 재귀 호출이 포함되지만, 프리미티브 호출 수는 초기 진입만 계산합니다. 총 호출 수/프리미티브 호출 수 형식으로 표시됩니다(예를 들어, 3/1 은 총 3회 호출, 프리미티브 호출 1회를 의미합니다).

호출자/피호출자 관계

어떤 함수가 특정 함수를 호출했는지(호출자) 그리고 해당 함수가 어떤 함수를 호출했는지(피호출자)를 보여줍니다. 플레임 그래프는 이를 중첩된 사각형으로 시각화할 수 있으며, pstats는 print_callers() 및 print_callees() 메서드를 통해 이를 표시할 수 있습니다.

레거시 호환성¶

하위 호환성을 위해, cProfile 모듈은 :mod:`profiling.tracing`의 별칭으로 계속 사용 가능합니다. `import cProfile`를 사용하는 기존 코드는 모든 향후 버전의 Python에서 수정 없이 계속 작동합니다.

버전 3.15부터 폐지됨: 순수 Python인 profile 모듈은 더 이상 사용되지 않으며 Python 3.17에서 제거될 예정입니다. 대신 profiling.tracing (또는 별칭인 cProfile)을 사용하십시오. 마이그레이션 방법은 :mod:`profile`을 참조하세요.
