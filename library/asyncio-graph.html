호출 그래프 인트로스펙션¶

소스 코드: Lib/asyncio/graph.py

asyncio는 실행 중인 coroutine 또는 task, 또는 중단된 future 전체 호출 그래프를 추적할 강력한 런타임 호출 그래프 인트로스펙션 유틸리티를 제공합니다. 이러한 유틸리티와 기반 메커니즘은 Python 프로그램 내부 또는 외부 프로파일러 및 디버거에서 사용할 수 있습니다.

Added in version 3.14.

asyncio.print_call_graph(future=None, /, *, file=None, depth=1, limit=None)¶

현재 태스크 또는 제공된 Task 또는 :class:`Future`에 대한 비동기 호출 그래프를 출력합니다.

이 함수는 최상위 프레임부터 호출 지점 쪽으로 내려가면서 항목을 인쇄합니다.

이 함수는 선택적 future 인자를 받습니다. 전달되지 않으면 현재 실행 중인 태스크가 사용됩니다.

함수가 현재 태스크*에 호출된 경우, 선택적 키워드 전용 *depth 인자를 사용하여 스택 맨 위에서 지정된 수만큼 프레임을 건너뛸 수 있습니다.

선택적 키워드 전용 limit 인자가 제공되면, 결과 그래프의 각 호출 스택은 최대 abs(limit) 개의 항목을 포함하도록 잘립니다. limit 이 양수이면 남은 항목은 호출 지점에 가장 가까운 항목입니다. limit 이 음수이면 가장 위쪽 항목이 남습니다. limit 이 생략되거나 None 이면 모든 항목이 표시됩니다. limit 이 0 이면 호출 스택은 전혀 인쇄되지 않고, “awaited by” 정보만 인쇄됩니다.

file 이 생략되거나 None 이면, 이 함수는 sys.stdout 에 출력합니다.

예제:

다음 Python 코드는:

import asyncio

async def test():
    asyncio.print_call_graph()

async def main():
    async with asyncio.TaskGroup() as g:
        g.create_task(test(), name='test')

asyncio.run(main())

를 인쇄할 것입니다:

* Task(name='test', id=0x1039f0fe0)
+ Call stack:
|   File 't2.py', line 4, in async test()
+ Awaited by:
   * Task(name='Task-1', id=0x103a5e060)
      + Call stack:
      |   File 'taskgroups.py', line 107, in async TaskGroup.__aexit__()
      |   File 't2.py', line 7, in async main()
asyncio.format_call_graph(future=None, /, *, depth=1, limit=None)¶

print_call_graph() 와 유사하지만 문자열을 반환합니다. future 가 None 이고 현재 태스크가 없으면 빈 문자열을 반환합니다.

asyncio.capture_call_graph(future=None, /, *, depth=1, limit=None)¶

현재 태스크 또는 제공된 Task 또는 :class:`Future`에 대한 비동기 호출 그래프를 캡처합니다.

이 함수는 선택적 future 인자를 받습니다. 전달되지 않으면 현재 실행 중인 태스크가 사용됩니다. 현재 태스크가 없으면 빈 None 을 반환합니다.

함수가 현재 태스크*에 호출된 경우, 선택적 키워드 전용 *depth 인자를 사용하여 스택 맨 위에서 지정된 수만큼 프레임을 건너뛸 수 있습니다.

FutureCallGraph 데이터 클래스 객체를 반환합니다:

  • FutureCallGraph(future, call_stack, awaited_by)

    여기서 future 는 Future 또는 Task 에 대한 참조입니다 (또는 그 서브클래스).

    call_stack``은 ``FrameCallGraphEntry 객체의 튜플입니다.

    awaited_by``는 ``FutureCallGraph 객체의 튜플입니다.

  • FrameCallGraphEntry(frame)

    여기서 frame 은 호출 스택에 있는 일반 Python 함수의 프레임 객체입니다.

저수준 유틸리티 함수¶

비동기 호출 그래프를 추적하려면 asyncio는 shield() 또는 TaskGroup`와 같은 제어 흐름 구조의 협력이 필요합니다. `:meth:`Future.add_done_callback() <asyncio.Future.add_done_callback>`와 같은 낮은 수준의 API를 가진 중간 :class:`Future 객체가 사용될 때마다, 다음과 같은 두 함수를 사용하여 asyncio에 이러한 중간 Future 객체가 래핑하거나 제어하는 태스크와 정확하게 어떻게 연결되어 있는지 알려야 합니다.

asyncio.future_add_to_awaited_by(future, waiter, /)¶

future 가 waiter 에 의해 기다려진 기록을 추가합니다.

future*와 *waiter 모두 Future 또는 Task 또는 이들의 서브 클래스 인스턴스여야 하며, 그렇지 않으면 호출에 아무런 효과가 없습니다.

future_add_to_awaited_by() 호출은 반드시 같은 인수를 사용하여 future_discard_from_awaited_by() 함수 호출로 이어져야 합니다.

asyncio.future_discard_from_awaited_by(future, waiter, /)¶

future 가 더 이상 waiter 에 의해 기다려지지 않도록 기록합니다.

future*와 *waiter 모두 Future 또는 Task 또는 이들의 서브 클래스 인스턴스여야 하며, 그렇지 않으면 호출에 아무런 효과가 없습니다.
