importlib.resources.abc\ – 리소스의 추상 기본 클래스¶

**소스 코드:** :source:`Lib/importlib/resources/abc.py`

Added in version 3.11.

class importlib.resources.abc.ResourceReader¶

TraversableResources 에 의해 사용 중단됨

resources 를 읽을 수 있는 기능을 제공하기 위한 추상 베이스 클래스 입니다.

이 ABC의 관점에서, resource 는 패키지 내에 배포되는 이진 아티팩트입니다. 일반적으로 이는 패키지의 __init__.py 파일 옆에 존재하는 데이터 파일과 같은 것입니다. 이 클래스의 목적은 패키지와 해당 데이터 파일(들)이 예를 들어 zip 파일에 저장되어 있는지 아니면 파일 시스템에 저장되어 있는지와 관계없이 그러한 데이터 파일에 접근하는 것을 추상화하는 데 도움을 주는 것입니다.

이 클래스의 모든 메서드에 대해, resource 인자는 개념적으로 단지 파일 이름을 나타내는 :term:`path-like object`여야 합니다. 이는 resource 인자에 하위 디렉터리 경로가 포함되어서는 안 된다는 것을 의미합니다. 이는 리더가 속한 패키지의 위치가 “디렉터리” 역할을 하기 때문입니다. 따라서 디렉터리와 파일 이름에 대한 은유는 각각 패키지와 리소스를 의미합니다. 이것이 이 클래스의 인스턴스가 여러 패키지나 모듈을 나타낼 가능성(대신)이 있는 것보다 특정 패키지에 직접 연관될 것으로 예상되는 이유이기도 합니다.

리소스 읽기를 지원하려는 로더는 이 ABC의 인터페이스를 구현하는 객체를 반환하는 get_resource_reader(fullname) 라는 메서드를 제공해야 합니다. 만약 fullname으로 지정된 모듈이 패키지가 아니라면, 이 메서드는 None 을 반환해야 합니다. 이 ABC와 호환되는 객체는 지정된 모듈이 패키지일 때만 반환되어야 합니다.

버전 3.12부터 폐지됨: 대신 :class:`importlib.resources.abc.TraversableResources`를 사용하십시오.

abstractmethod open_resource(resource)¶

resource 의 이진 읽기를 위한 열린, file-like object 를 반환합니다.

리소스를 찾을 수 없으면 :exc:`FileNotFoundError`가 발생합니다.

abstractmethod resource_path(resource)¶

resource 에 대한 파일 시스템 경로를 반환합니다.

리소스가 파일 시스템에 구체적으로 존재하지 않으면 :exc:`FileNotFoundError`를 발생시킵니다.

abstractmethod is_resource(path)¶

명명된 path 가 리소스로 간주되면 True 를 반환합니다. path 가 존재하지 않으면 FileNotFoundError 가 발생합니다.

버전 3.10에서 변경: name 인자는 path 로 이름이 변경되었습니다.

abstractmethod contents()¶

패키지 내용을 포함하는 문자열의 :term:`iterable`을 반환합니다. 반복자가 반환하는 모든 이름이 실제 리소스일 필요는 없다는 점에 유의하십시오. 예를 들어, :meth:`is_resource`가 false를 반환할 이름도 반환하는 것은 허용됩니다.

리소스가 아닌 이름을 반환하는 것은 패키지와 그 리소스가 어떻게 저장되는지 사전에 알려져 있고, 해당 리소스가 아닌 이름들이 유용할 수 있는 상황을 허용하기 위함입니다. 예를 들어, 서브디렉터리 이름을 반환하는 것이 허용되는데, 이는 패키지와 리소스가 파일 시스템에 저장되어 있다는 것을 알 때 해당 서브디렉터리 이름들을 직접 사용할 수 있도록 하기 위함입니다.

추상 메서드는 항목이 없는 이터러블을 반환합니다.

class importlib.resources.abc.Traversable¶

디렉터리를 순회하고 파일을 여는 데 적합한 pathlib.Path 메서드의 부분 집합을 포함하는 객체입니다.

파일 시스템 상에 객체를 표현하려면 :meth:`importlib.resources.as_file`을 사용하십시오.

name¶

부모 참조가 없는 이 객체의 기본 이름입니다.

abstractmethod iterdir()¶

self에서 순회 가능한 객체를 산출합니다.

abstractmethod is_dir()¶

self가 디렉터리인지 여부를 True 로 반환합니다.

abstractmethod is_file()¶

self가 파일인지 여부를 True 로 반환합니다.

abstractmethod joinpath(*pathsegments)¶

pathsegments 에 따라 디렉터리를 순회하고 결과를 Traversable 로 반환합니다.

각 pathsegments 인자는 슬래시(/, posixpath.sep)로 구분된 여러 이름을 포함할 수 있습니다. 예를 들어, 다음은 같습니다:

files.joinpath('subdir', 'subsuddir', 'file.txt')
files.joinpath('subdir/subsuddir/file.txt')

일부 Traversable 구현체는 프로토콜의 최신 버전으로 업데이트되지 않았을 수 있다는 점에 주의하십시오. 이러한 구현체와의 호환성을 위해 joinpath 호출 시 경로 구분 기호 없이 단일 인수를 제공하십시오. 예를 들면:

files.joinpath('subdir').joinpath('subsubdir').joinpath('file.txt')

버전 3.11에서 변경: joinpath 는 여러 pathsegments 를 허용하며, 이 세그먼트에는 경로 구분 기호로 슬래시가 포함될 수 있습니다. 이전에는 단일 child 인수만 허용되었습니다.

abstractmethod __truediv__(child)¶

self에서 순회 가능한 자식을 반환합니다. 이는 joinpath(child) 와 동등합니다.

abstractmethod open(mode='r', *args, **kwargs)¶

mode 는 텍스트로 열려면 ‘r’ 또는 이진으로 열려면 ‘rb’일 수 있습니다. 읽기에 적합한 핸들( pathlib.Path.open 과 동일)을 반환합니다.

텍스트로 열 경우, :class:`io.TextIOWrapper`에서 허용하는 인코딩 매개변수를 받습니다.

read_bytes()¶

self의 내용을 바이트로 읽습니다.

read_text(encoding=None)¶

self의 내용을 텍스트로 읽습니다.

class importlib.resources.abc.TraversableResources¶

importlib.resources.files() 인터페이스를 제공할 수 있는 리소스 리더를 위한 추상 베이스 클래스입니다. 서브 클래스는 ResourceReader`를 서브클래싱하고 :class:!ResourceReader`의 추상 메서드에 대한 구체적인 구현을 제공합니다. 따라서, TraversableResources`를 제공하는 모든 로더는 :class:!ResourceReader`도 제공합니다.

리소스 읽기를 지원하려는 로더는 이 인터페이스를 구현해야 합니다.

abstractmethod files()¶

로딩된 패키지에 대한 importlib.resources.abc.Traversable 객체를 반환합니다.
