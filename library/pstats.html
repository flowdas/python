pstats — 프로파일러 통계¶

소스 코드: Lib/pstats.py

pstats 모듈은 파이썬 profiler가 생성한 프로파일링 통계를 읽고, 조작하고, 표시하는 도구를 제공합니다. 이 모듈은 profiling.tracing (결정론적 profiler)와 profiling.sampling (통계적 profiler)의 출력을 모두 읽습니다.

프로파일 데이터 읽기 및 표시¶

Stats 클래스는 프로파일 데이터 작업을 위한 주요 인터페이스입니다. 파일에서 통계를 읽거나 Profile 객체에서 직접 읽을 수 있습니다.

파일에서 통계를 불러와 기본 보고서를 인쇄하기:

import pstats

p = pstats.Stats('profile_output.prof')
p.print_stats()

Stats 객체는 데이터를 인쇄하기 전에 정렬하고 필터링하는 메서드를 제공합니다. 예를 들어, 누적 시간이 가장 높은 상위 10개 함수를 보려면:

from pstats import SortKey

p = pstats.Stats('profile_output.prof')
p.sort_stats(SortKey.CUMULATIVE).print_stats(10)

통계 작업¶

Stats 클래스는 메서드 체이닝을 지원하여 여러 작업을 수행하기 편리합니다:

p = pstats.Stats('restats')
p.strip_dirs().sort_stats(-1).print_stats()

strip_dirs() 메서드는 파일 이름에서 디렉터리 경로를 제거하여 출력을 더 간결하게 만듭니다. sort_stats() 메서드는 정렬 순서를 제어하기 위해 다양한 키를 받습니다.

다양한 정렬 키는 성능의 다른 측면을 강조합니다:

from pstats import SortKey

# 누적 시간이 가장 많은 함수
p.sort_stats(SortKey.CUMULATIVE).print_stats(10)

# 자체 코드에서 가장 많은 시간을 소비하는 함수
p.sort_stats(SortKey.TIME).print_stats(10)

# 이름으로 정렬된 함수
p.sort_stats(SortKey.NAME).print_stats()

필터링 결과¶

print_stats() 메서드는 표시할 함수를 필터링하는 제한 조건을 받습니다. 제한 조건은 정수(개수 제한), 부동 소수점(0과 1 사이의 백분율 선택), 또는 문자열(정규 표현식으로 함수 이름 일치)일 수 있습니다.

상위 10%만 인쇄:

p.print_stats(.1)

이름에 “init”을 포함하는 함수만 인쇄:

p.print_stats('init')

제한 사항 결합 (순차적으로 적용됨):

# 상위 10%, 그 다음 "init"을 포함하는 함수만
p.print_stats(.1, 'init')

# "foo:"와 일치하는 파일의 함수를 상위 50%로 제한
p.sort_stats(SortKey.FILENAME).print_stats('foo:', .5)

호출 관계 분석¶

print_callers() 메서드는 각 표시된 함수를 어떤 함수가 호출했는지 보여줍니다:

p.print_callers()

print_callees() 메서드는 반대 관계를 보여주며, 각 표시된 함수가 어떤 함수를 호출했는지 나열합니다:

p.print_callees()

두 메서드 모두 :meth:`~Stats.print_stats`와 동일한 제한 인자를 받습니다.

여러 프로파일 결합¶

다양한 프로파일링 실행의 통계를 단일 Stats 객체로 결합할 수 있습니다:

# 여러 파일을 한 번에 로드
p = pstats.Stats('run1.prof', 'run2.prof', 'run3.prof')

# 또는 파일을 증분하여 추가
p = pstats.Stats('run1.prof')
p.add('run2.prof')
p.add('run3.prof')

파일이 결합될 때, 동일한 함수(같은 파일, 줄, 이름)에 대한 통계가 누적되어 모든 프로파일링 실행에 걸친 종합적인 뷰를 제공합니다.

Stats 클래스¶

class pstats.Stats(*filenames_or_profile, stream=sys.stdout)¶

프로파일 데이터로부터 통계 객체를 생성합니다.

인자는 파일 이름(문자열 또는 경로와 같은 객체)이거나 Profile 객체일 수 있습니다. 여러 출처가 제공되면, 해당 통계들이 결합됩니다.

stream 인자는 print_stats() 및 관련 메서드의 출력이 기록될 위치를 지정하며, 기본값은 :data:`sys.stdout`입니다.

프로파일 데이터 형식은 이를 생성한 Python 버전에 특화되어 있습니다. Python 버전 간 또는 서로 다른 프로파일러 간에는 호환성 보장이 없습니다.

strip_dirs()¶

모든 파일 이름에서 선행 경로 정보를 제거합니다.

이 메서드는 객체를 제자리에서 수정하고 메서드 체이닝을 위해 객체를 반환합니다. 제거 후에는 통계가 무작위 순서로 간주됩니다.

제거 과정에서 두 함수가 구별할 수 없게 되면 (같은 파일 이름, 줄 번호, 함수 이름), 해당 통계는 단일 항목으로 합산됩니다.

add(*filenames)¶

추가 파일로부터 프로파일링 데이터 추가합니다.

파일들은 동일한 프로파일러 유형에 의해 생성되었어야 합니다. 동일한 함수의 통계는 누적됩니다.

dump_stats(filename)¶

현재 통계를 파일에 저장합니다.

파일이 존재하지 않으면 생성되고, 이미 존재하면 덮어씁니다. 저장된 데이터는 새 Stats 객체를 생성하여 로드할 수 있습니다.

sort_stats(*keys)¶

지정된 기준에 따라 통계를 정렬합니다.

각 키는 문자열이거나 SortKey 열거형 멤버일 수 있습니다. 여러 키가 제공될 경우, 나중 키가 이전 키의 동점 처리 기준을 변경합니다.

SortKey 열거형 멤버를 사용하는 것이 문자열보다 권장되는데, 더 나은 오류 검사를 제공하기 때문입니다:

from pstats import SortKey
p.sort_stats(SortKey.CUMULATIVE)

통계의 모든 정렬은 내림차순(가장 시간이 많이 소요되는 항목을 먼저)이며, 이름, 파일 및 줄 번호 정렬은 오름차순(알파벳순)입니다.

SortKey.NFL 과 SortKey.STDNAME 의 차이점은 NFL이 줄 번호를 숫자로 정렬하는 반면, STDNAME은 문자열로 정렬한다는 것입니다. sort_stats(SortKey.NFL) 은 sort_stats(SortKey.NAME, SortKey.FILENAME, SortKey.LINE) 과 같습니다.

호환성 때문에, 숫자 인자 -1, 0, 1, 및 2 도 허용되며, 이는 각각 'stdname', 'calls', 'time', 및 'cumulative' 을 의미합니다.

Added in version 3.7: SortKey 열거형입니다.

reverse_order()¶

현재 정렬 순서를 반전합니다.

기본적으로, 정렬 방향은 정렬 키에 맞게 적절하게 선택됩니다(시간 기반 키는 내림차순, 이름 기반 키는 오름차순). 이 메서드는 그 선택을 반전합니다.

print_stats(*restrictions)¶

프로파일링 통계 보고서를 인쇄합니다.

출력에는 데이터를 요약하는 헤더 라인이 포함되며, 그 뒤에 마지막 sort_stats() 호출에 따라 정렬된 함수 통계표가 따릅니다.

제약 사항은 출력을 필터링합니다. 각 제약 사항은 다음 중 하나입니다:

  • 정수: 출력을 해당 개수만큼으로 제한합니다.

  • 0.0과 1.0 사이의 float: 해당 비율의 항목을 선택합니다.

  • 문자열: 정규식을 사용하여 함수 이름을 일치시킵니다.

제약 사항은 순차적으로 적용됩니다. 예를 들어:

print_stats(.1, 'foo:')

먼저 상위 10%로 제한한 다음, ‘foo:’와 일치하는 함수로 필터링합니다.

print_callers(*restrictions)¶

통계에서 각 함수의 호출자를 인쇄합니다.

필터링된 결과의 각 함수에 대해, 어떤 함수가 이 함수를 호출했는지, 그리고 얼마나 자주 호출했는지 보여줍니다.

profiling.tracing (또는 cProfile)를 사용하면, 각 호출자 라인은 세 개의 숫자를 보여줍니다: 해당 호출자로부터의 호출 횟수, 그리고 해당 특정 호출에 대한 총 시간 및 누적 시간입니다.

:meth:`print_stats`와 동일한 제약 인수를 받습니다.

print_callees(*restrictions)¶

통계에서 각 함수가 호출한 함수를 인쇄합니다.

이는 :meth:`print_callers`의 역순으로, 나열된 각 함수가 어떤 함수를 호출했는지를 보여줍니다.

:meth:`print_stats`와 동일한 제약 인수를 받습니다.

get_stats_profile()¶

통계를 포함하는 StatsProfile 객체를 반환합니다.

반환된 객체는 함수 이름을 타이밍 및 호출 횟수 정보를 포함하는 FunctionProfile 객체로 매핑하여 프로파일 데이터에 대한 프로그래밍 방식 접근을 제공합니다.

Added in version 3.9.

class pstats.SortKey¶

:meth:`Stats.sort_stats`에 대한 유효한 정렬 키의 열거형입니다.

CALLS¶

호출 횟수별로 정렬합니다.

CUMULATIVE¶

누적 시간별로 정렬합니다.

FILENAME¶

파일 이름별로 정렬합니다.

LINE¶

줄 번호별로 정렬합니다.

NAME¶

함수 이름별로 정렬합니다.

NFL¶

이름, 그런 다음 파일, 그런 다음 줄 번호로 정렬합니다(숫자 줄 정렬).

PCALLS¶

기본(비재귀) 호출 횟수별로 정렬합니다.

STDNAME¶

표준 이름(문자열 기반 줄 정렬)으로 정렬합니다.

TIME¶

내부 시간(하위 호출을 제외한 함수 내 시간)으로 정렬합니다.

명령줄 인터페이스¶

pstats 모듈은 프로파일 데이터를 대화식으로 탐색하는 스크립트로 호출될 수 있습니다:

python -m pstats profile_output.prof

이것은 통계를 검사하기 위한 라인 지향 인터페이스( cmd 를 기반으로 구축됨)를 엽니다. 사용 가능한 명령을 원하시면 프롬프트에 help 를 입력하세요.
