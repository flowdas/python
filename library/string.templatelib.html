템플릿 문자열¶

Added in version 3.14.

템플릿 문자열은 사용자 정의 문자열 처리를 위한 메커니즘입니다. Python의 f-strings`의 모든 유연성을 가지지만, 결합되기 *전에* 문자열의 정적 부분과 보간된 부분에 액세스할 수 있는 :class:`Template 인스턴스를 반환합니다.

t-문자열을 작성하려면 'f' 대신 't' 접두사를 사용합니다. 예를 들어:

>>> pi = 3.14
>>> t't-strings are new in Python {pi!s}!'
Template(
   strings=('t-strings are new in Python ', '!'),
   interpolations=(Interpolation(3.14, 'pi', 's', ''),)
)

타입¶

class string.templatelib.Template¶

Template 클래스는 템플릿 문자열의 내용을 설명합니다. 이 클래스는 불변이어서 템플릿의 속성을 다시 할당할 수 없습니다.

Template 인스턴스를 생성하는 가장 일반적인 방법은 <t-strings> 템플릿 문자열 리터럴 구문을 사용하는 것입니다. 이 구문은 f-strings <f-strings>`와 동일하지만, ``f` 대신 t 접두사를 사용합니다:

>>> cheese = 'Red Leicester'
>>> template = t"We're fresh out of {cheese}, sir."
>>> type(template)
<class 'string.templatelib.Template'>

템플릿은 리터럴 values 속성은 보간된 값들을 담고 있습니다:

>>> cheese = 'Camembert'
>>> template = t'Ah! We do have {cheese}.'
>>> template.strings
('Ah! We do have ', '.')
>>> template.interpolations
(Interpolation('Camembert', ...),)
>>> template.values
('Camembert',)

strings 튜플은 interpolations 및 :attr:`!values`보다 요소가 하나 더 많으며; 보간 구문은 문자열 사이에 “속합니다”. 튜플이 정렬되면 이해하기 쉬울 수 있습니다.

template.strings:  ('Ah! We do have ',              '.')
template.values:   (                   'Camembert',    )

속성

strings: tuple[str, ...]¶

템플릿의 정적 문자열로 구성된 tuple.

>>> cheese = 'Camembert'
>>> template = t'Ah! We do have {cheese}.'
>>> template.strings
('Ah! We do have ', '.')

빈 문자열도 튜플에 포함됩니다:

>>> response = 'We do have '
>>> cheese = 'Camembert'
>>> template = t'Ah! {response}{cheese}.'
>>> template.strings
('Ah! ', '', '.')

빈 strings 튜플은 결코 비어 있지 않으며, 항상 interpolations``와 ``values 튜플보다 하나의 문자열을 더 포함합니다:

>>> t''.strings
('',)
>>> t''.values
()
>>> t'{'cheese'}'.strings
('', '')
>>> t'{'cheese'}'.values
('cheese',)
interpolations: tuple[Interpolation, ...]¶

템플릿에 포함된 보간(interpolations)들의 :class:`tuple`입니다.

>>> cheese = 'Camembert'
>>> template = t'Ah! We do have {cheese}.'
>>> template.interpolations
(Interpolation('Camembert', 'cheese', None, ''),)

interpolations 튜플은 비어 있을 수 있으며, 항상 strings 튜플보다 값이 하나 적습니다:

>>> t'Red Leicester'.interpolations
()
values: tuple[object, ...]¶

템플릿의 모든 보간된 값들의 튜플입니다.

>>> cheese = 'Camembert'
>>> template = t'Ah! We do have {cheese}.'
>>> template.values
('Camembert',)

values 튜플은 항상 interpolations 튜플과 길이가 같습니다. 항상 tuple(i.value for i in template.interpolations) 와 동등합니다.

메서드

__new__(*args: str | Interpolation)¶

리터럴 구문이 :class:`!Template`을 생성하는 가장 일반적인 방법이지만, 생성자를 사용하여 직접 생성하는 것도 가능합니다:

>>> from string.templatelib import Interpolation, Template
>>> cheese = 'Camembert'
>>> template = Template(
...     'Ah! We do have ', Interpolation(cheese, 'cheese'), '.'
... )
>>> list(template)
['Ah! We do have ', Interpolation('Camembert', 'cheese', None, ''), '.']

여러 문자열이 연속적으로 전달되면, 이들은 strings 어트리뷰트의 단일 값으로 연결됩니다. 예를 들어, 다음 코드는 단일 최종 문자열을 가진 :class:`Template`을 생성합니다:

>>> from string.templatelib import Template
>>> template = Template('Ah! We do have ', 'Camembert', '.')
>>> template.strings
('Ah! We do have Camembert.',)

여러 간격(interpolations)이 연속적으로 전달되면, 이들은 개별 간격으로 처리되며 그 사이에 빈 문자열이 삽입됩니다. 예를 들어, 다음 코드는 strings 어트리뷰트에 빈 플레이스홀더를 가진 템플릿을 생성합니다:

>>> from string.templatelib import Interpolation, Template
>>> template = Template(
...     Interpolation('Camembert', 'cheese'),
...     Interpolation('.', 'punctuation'),
... )
>>> template.strings
('', '', '')
iter(template)

템플릿을 반복하여, 각 비어있지 않은 문자열과 :class:`Interpolation`을 올바른 순서로 생성합니다:

>>> cheese = 'Camembert'
>>> list(t'Ah! We do have {cheese}.')
['Ah! We do have ', Interpolation('Camembert', 'cheese', None, ''), '.']

조심

비어있는 문자열은 반복에 포함되지 않습니다:

>>> response = 'We do have '
>>> cheese = 'Camembert'
>>> list(t'Ah! {response}{cheese}.')
['Ah! ',
 Interpolation('We do have ', 'response', None, ''),
 Interpolation('Camembert', 'cheese', None, ''),
 '.']
template + other
template += other

이 템플릿을 다른 템플릿과 연결하고, 새로운 Template 인스턴스를 반환합니다:

>>> cheese = 'Camembert'
>>> list(t'Ah! ' + t'We do have {cheese}.')
['Ah! We do have ', Interpolation('Camembert', 'cheese', None, ''), '.']

Template`과 ``str``을 연결하는 것은 **지원되지 않습니다**. 이는 문자열을 정적 문자열로 처리해야 하는지, 아니면 간격(interpolation)으로 처리해야 하는지 불분명하기 때문입니다. :class:!Template`을 문자열과 연결하려면, 문자열을 Template`에 직접 래핑하거나(정적 문자열로 처리할 경우) :class:!Interpolation`을 사용해야 합니다(동적으로 처리할 경우):

>>> from string.templatelib import Interpolation, Template
>>> template = t'Ah! '
>>> # Treat 'We do have ' as a static string
>>> template += Template('We do have ')
>>> # Treat cheese as an interpolation
>>> cheese = 'Camembert'
>>> template += Template(Interpolation(cheese, 'cheese'))
>>> list(template)
['Ah! We do have ', Interpolation('Camembert', 'cheese', None, '')]
class string.templatelib.Interpolation¶

Interpolation 타입은 템플릿 문자열 내부의 표현식을 나타냅니다. 불변(immutable)하므로, 보간의 속성은 다시 할당할 수 없습니다.

보간은 패턴 매칭을 지원하여, :ref:`match statement <match>`를 사용하여 속성과 비교할 수 있게 합니다:

>>> from string.templatelib import Interpolation
>>> interpolation = t'{1. + 2.:.2f}'.interpolations[0]
>>> interpolation
Interpolation(3.0, '1. + 2.', None, '.2f')
>>> match interpolation:
...     case Interpolation(value, expression, conversion, format_spec):
...         print(value, expression, conversion, format_spec, sep=' | ')
...
3.0 | 1. + 2. | None | .2f

보간은 값의 타입을 따라 :ref:`generic <generics>`입니다.

속성

value: object¶

보간의 평가된 값입니다.

>>> t'{1 + 2}'.interpolations[0].value
3
expression: str¶

t-문자열 리터럴로 생성된 보간의 경우, expression`은 중괄호(``{` & }) 내에서 발견되는 표현식 텍스트, 공백을 포함하며, 중괄호 자체는 제외하고, 그리고 !, :, 또는 =``가 존재하는 경우 가장 첫 번째 ``!, :, 또는 = 앞에 끝나는 텍스트입니다. 수동으로 생성된 보간의 경우, :attr:`!expression`은 보간 인스턴스를 구성할 때 제공되는 임의의 문자열입니다.

런타임에 강제되지는 않지만, 수동으로 생성된 Interpolation 인스턴스의 expression 필드에는 유효한 Python 표현식이나 빈 문자열을 사용하는 것을 권장합니다.

>>> t'{1 + 2}'.interpolations[0].expression
'1 + 2'
conversion: Literal['a', 'r', 's'] | None¶

값에 적용할 변환, 또는 None 입니다.

:attr:`!conversion`은 값에 적용할 선택적 변환입니다:

>>> t'{1 + 2!a}'.interpolations[0].conversion
'a'

참고

변환이 자동으로 적용되는 f-문자열과 달리, t-문자열의 예상되는 동작은 Template`을 *처리하는* 코드가 :attr:!conversion`을 어떻게 해석하고 적용할지 결정하는 것입니다. 편의를 위해, convert() 함수를 사용하여 f-문자열 변환 의미론을 모방할 수 있습니다.

format_spec: str¶

값에 적용할 포맷 지정자입니다.

:attr:`!format_spec`은 값을 표시하는 포맷 지정자로 사용되는 선택적이며 임의의 문자열입니다:

>>> t'{1 + 2:.2f}'.interpolations[0].format_spec
'.2f'

참고

포맷 지정자가 format() 프로토콜을 통해 자동으로 적용되는 f-문자열과 달리, t-문자열의 예상되는 동작은 보간을 처리하는 코드가 포맷 지정자를 어떻게 해석하고 적용할지 결정한다는 것입니다. 결과적으로, 보간에 있는 format_spec 값은 format() 프로토콜을 준수하지 않는 문자열을 포함하여 임의의 문자열일 수 있습니다.

메서드

__new__(value: object, expression: str, conversion: Literal['a', 'r', 's'] | None = None, format_spec: str = '')¶

구성 요소들로부터 새로운 Interpolation 객체를 생성합니다.

매개변수:

  • value – 범위 내에서 평가된 보간의 결과입니다.

  • expression – 유효한 Python 표현식의 텍스트, 또는 빈 문자열입니다.

  • conversion – 사용하려는 conversion <formatstrings>`입니다. ``None`, 'a', 'r' 또는 's' 중 하나입니다.

  • format_spec – 값을 표시하는 데 사용되는 선택적의 임의의 문자열이며, format specification 역할을 합니다.

도우미 함수¶

string.templatelib.convert(obj, /, conversion)¶

주어진 객체 obj 에 형식화된 문자열 리터럴 conversion 의미를 적용합니다. 이는 사용자 정의 템플릿 문자열 처리 로직에 매우 유용합니다.

현재 세 가지 변환 플래그가 지원됩니다:

  • 's'``는 값에 대해 :func:`str`을 호출합니다 (!s``와 유사).

  • 'r'``은 :func:`repr`을 호출하며 (!r``와 유사), 그리고

  • 'a'``는 :func:`ascii`를 호출합니다 (!a``와 유사).

변환 플래그가 None 이면, obj 는 변경되지 않은 상태로 반환됩니다.
