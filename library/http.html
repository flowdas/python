http — HTTP 모듈¶

소스 코드: Lib/http/__init__.py

:mod:`!http`는 하이퍼텍스트 전송 프로토콜로 작업하기 위한 여러 모듈을 수집하는 패키지입니다:

  • http.client는 저수준 HTTP 프로토콜 클라이언트입니다. 고수준의 URL 열기는 urllib.request를 사용합니다

  • http.server는 socketserver에 기반을 둔 기본적인 HTTP 서버 클래스를 포함합니다

  • http.cookies는 쿠키를 사용하여 상태 관리를 구현하는 유틸리티가 있습니다

  • http.cookiejar는 쿠키의 지속성을 제공합니다

또한 http 모듈은 http 관련 코드 작업에 도움이 되는 다음 열거형들을 정의합니다:

class http.HTTPStatus¶

Added in version 3.5.

HTTP 상태 코드, 이유 구문 그리고 긴 영문 설명의 집합을 정의하는 enum.IntEnum의 서브 클래스입니다.

사용법:

>>> from http import HTTPStatus
>>> HTTPStatus.OK
HTTPStatus.OK
>>> HTTPStatus.OK == 200
True
>>> HTTPStatus.OK.value
200
>>> HTTPStatus.OK.phrase
'OK'
>>> HTTPStatus.OK.description
'Request fulfilled, document follows'
>>> list(HTTPStatus)
[HTTPStatus.CONTINUE, HTTPStatus.SWITCHING_PROTOCOLS, ...]

이전 버전과의 호환성을 유지하기 위해 열거값은 http.client 모듈에 상수 형태로도 있습니다. 열거명과 상수명은 동일합니다 (즉, http.HTTPStatus.OK는 http.client.OK로도 사용 가능합니다).

버전 3.7에서 변경: 421 MISDIRECTED_REQUEST 상태 코드 추가.

Added in version 3.8: 451 UNAVAILABLE_FOR_LEGAL_REASONS 상태 코드가 추가.

Added in version 3.9: 103 EARLY_HINTS, 418 IM_A_TEAPOT 및 425 TOO_EARLY 상태 코드가 추가되었습니다.

버전 3.13에서 변경: 상태 상수 이름에 RFC9110 명명 규칙을 구현했습니다. 이전 상수 이름은 하위 호환성을 위해 유지됩니다: 413 REQUEST_ENTITY_TOO_LARGE, 414 REQUEST_URI_TOO_LONG, 416 REQUESTED_RANGE_NOT_SATISFIABLE, 및 422 UNPROCESSABLE_ENTITY.

>>> from http import HTTPStatus
>>> HTTPStatus.OK.is_success

>>> HTTPStatus.OK.is_client_error

class http.HTTPMethod¶

Added in version 3.11.

HTTP 메서드와 영문 설명의 집합을 정의하는 enum.StrEnum의 서브 클래스입니다.

사용법:

>>> from http import HTTPMethod
>>>
>>> HTTPMethod.GET
<HTTPMethod.GET>
>>> HTTPMethod.GET == 'GET'
True
>>> HTTPMethod.GET.value
'GET'
>>> HTTPMethod.GET.description
'Retrieve the target.'
>>> list(HTTPMethod)
[<HTTPMethod.CONNECT>,
 <HTTPMethod.DELETE>,
 <HTTPMethod.GET>,
 <HTTPMethod.HEAD>,
 <HTTPMethod.OPTIONS>,
 <HTTPMethod.PATCH>,
 <HTTPMethod.POST>,
 <HTTPMethod.PUT>,
 <HTTPMethod.TRACE>]
