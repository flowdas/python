compression.zstd — Zstandard 형식과 호환되는 압축¶

Added in version 3.14.

소스 코드: Lib/compression/zstd/__init__.py

이 모듈은 Zstandard (또는 zstd) 압축 알고리즘을 사용한 데이터의 압축 및 압축 해제에 대한 클래스와 함수를 제공합니다. zstd manual <https://facebook.github.io/zstd/doc/api_manual_latest.html>`__에서 Zstandard를 “실시간 압축 시나리오를 목표로 하며 zlib 수준보다 더 낫고 효율적인 빠른 무손실 압축 알고리즘”으로 설명합니다. 또한, :program:`zstd 유틸리티가 생성한 .zst 파일과 원시 zstd 압축 스트림의 내용을 읽거나 쓰는 것을 지원하는 파일 인터페이스도 포함되어 있습니다.

compression.zstd 모듈에는 다음이 포함됩니다:

  • 압축된 파일을 읽고 쓰기 위한 open() 함수와 ZstdFile 클래스입니다.

  • 증분 압축(해제)을 위한 ZstdCompressor`와 :class:`ZstdDecompressor 클래스입니다.

  • 일회성 (압축/해제)을 위한 compress() 및 decompress() 함수입니다.

  • Zstandard 딕셔너리를 학습하고 관리하기 위한 train_dict() 및 finalize_dict() 함수와 ZstdDict 클래스입니다.

  • 고급 (압축/해제) 매개변수를 설정하기 위한 CompressionParameter, DecompressionParameter, 및 Strategy 클래스입니다.

This is an optional module. If it is missing from your copy of CPython, look for documentation from your distributor (that is, whoever provided Python to you). If you are the distributor, see Requirements for optional modules.

예외¶

exception compression.zstd.ZstdError¶

압축 파일 읽기 및 쓰기¶

compression.zstd.open(file, /, mode='rb', *, level=None, options=None, zstd_dict=None, encoding=None, errors=None, newline=None)¶

Zstandard 압축 파일을 바이너리 또는 텍스트 모드로 열고, :term:`파일 객체 <file object>`를 반환합니다.

file 인자는 파일 이름(str, bytes 또는 경로류 객체로 제공되는 경우)일 수 있으며, 이 경우 명명된 파일을 열거나 읽거나 쓸 기존 파일 객체일 수 있습니다.

mode 인자는 읽기용 'rb' (기본값), 덮어쓰기용 'wb', 추가용 'ab', 또는 배타적 생성을 위한 'xb' 일 수 있습니다. 이들은 각각 'r', 'w', 'a', 및 'x' 로도 동등하게 제공될 수 있습니다. 또한 각각 'rt', 'wt', 'at', 그리고 'xt' 를 사용하여 텍스트 모드로 열 수도 있습니다.

읽을 때 options 인자는 고급 압축 해제 매개변수를 제공하는 딕셔너리일 수 있습니다. 지원되는 매개변수에 대한 자세한 정보는 DecompressionParameter`를 참조하십시오. *zstd_dict* 인자는 압축 해제 중에 사용될 :class:`ZstdDict 인스턴스입니다. 읽을 때, level 인자가 None이 아니면 :exc:`!TypeError`가 발생합니다.

쓸 때 options 인자는 고급 압축 매개변수를 제공하는 딕셔너리일 수 있습니다. 지원되는 매개변수에 대한 자세한 정보는 CompressionParameter`를 참조하십시오. *level* 인자는 압축 데이터 작성 시 사용할 압축 레벨입니다. *level* 또는 *options* 중 하나만 None이 아닐 수 있습니다. *zstd_dict* 인자는 압축 중에 사용될 :class:`ZstdDict 인스턴스입니다.

바이너리 모드에서 이 함수는 ZstdFile 생성자와 동등합니다: ZstdFile(file, mode, ...). 이 경우, encoding, errors, 및 newline 매개변수는 제공하지 않아야 합니다.

텍스트 모드에서는 ZstdFile 객체가 생성되고 지정된 인코딩, 에러 처리 동작 및 줄 끝으로 io.TextIOWrapper 인스턴스로 감싸집니다.

class compression.zstd.ZstdFile(file, /, mode='rb', *, level=None, options=None, zstd_dict=None)¶

Zstandard 압축 파일을 바이너리 모드로 엽니다.

ZstdFile`은 이미 열려있는 :term:`파일 객체 <file object>`를 래핑하거나, 명명된 파일에 직접 작용할 수 있습니다. *file* 인자는 랩핑할 파일 객체 또는 열 파일의 이름(다음과 같은 :class:`str, bytes, 또는 경로류 객체)을 지정합니다. 기존 파일 객체를 래핑하는 경우, 래핑된 파일은 :class:`ZstdFile`이 닫혀도 닫히지 않습니다.

mode 인자는 읽기를 위한 'rb' (기본값), 덮어쓰기를 위한 'wb', 배타적 생성을 위한 'xb' 또는 추가를 위한 'ab' 중 하나일 수 있습니다. 이 값들은 각각 'r', 'w', 'x' 및 'a' 와도 동등하게 제공될 수 있습니다.

file 이 (실제 파일 이름 대신) 파일 객체인 경우, 'w' 모드는 파일을 자르지 않으며, 대신 'a' 와 동등합니다.

읽을 때 options 인자는 고급 압축 해제 매개변수를 제공하는 딕셔너리일 수 있습니다. 지원되는 매개변수에 대한 자세한 정보는 DecompressionParameter`를 참조하십시오. *zstd_dict* 인자는 압축 해제 중에 사용될 :class:`ZstdDict 인스턴스입니다. 읽을 때, level 인자가 None이 아니면 :exc:`!TypeError`가 발생합니다.

쓰기 시에 options 인자는 고급 압축 매개변수를 제공하는 딕셔너리일 수 있습니다. 지원되는 매개변수에 대한 자세한 정보는 CompressionParameter`를 참조하십시오. *level* 인자는 압축 데이터를 쓸 때 사용할 압축 수준입니다. *level* 또는 *options* 중 하나만 전달할 수 있습니다. *zstd_dict* 인자는 압축 중에 사용되는 :class:`ZstdDict 인스턴스입니다.

ZstdFile`는 :meth:`~io.BufferedIOBase.detach`와 :meth:`~io.IOBase.truncate`를 제외하고, :class:`io.BufferedIOBase`가 지정하는 모든 멤버들을 지원합니다. 이터레이션과 :keyword:`with 문을 사용할 수 있습니다.

다음 메서드와 어트리뷰트도 제공됩니다:

peek(size=-1)¶

파일 위치를 전진시키지 않고 버퍼링된 데이터를 반환합니다. EOF에 도달하지 않은 한, 최소한 1바이트의 데이터가 반환됩니다. 반환되는 정확한 바이트 수는 지정되지 않습니다 (size 인자는 무시됩니다).

참고

peek() 를 호출해도 ZstdFile 의 파일 위치는 변경되지 않지만, 하부 파일 객체의 위치가 바뀔 수 있습니다 (예로, ZstdFile 이 file 을 위한 파일 객체를 전달하여 생성된 경우).

mode¶

읽기 시 'rb', 그리고 쓰기 시 'wb' 를 사용합니다.

name¶

The name of the Zstandard file. Equivalent to the name attribute of the underlying file object.

메모리에서 데이터 압축 및 압축 해제¶

compression.zstd.compress(data, level=None, options=None, zstd_dict=None)¶

data (bytes-like object)를 압축하여, 압축된 데이터를 bytes 객체로 반환합니다.

level 인자는 압축 수준을 제어하는 정수입니다. level*은 *options*에서 :attr:`CompressionParameter.compression_level`을 설정하는 대안이 됩니다. *level*에 전달할 수 있는 값을 얻으려면 :attr:`~CompressionParameter.compression_level`에 대해 :meth:`~CompressionParameter.bounds`를 사용하십시오. 고급 압축 옵션이 필요한 경우, *level 인자를 생략하고 options 딕셔너리에서 CompressionParameter.compression_level 매개변수를 설정해야 합니다.

options 인자는 고급 압축 매개변수를 포함하는 Python 딕셔너리입니다. 압축 매개변수에 대한 유효한 키와 값은 CompressionParameter 문서의 일부로 설명되어 있습니다.

zstd_dict 인자는 압축 효율성을 개선하기 위해 훈련된 데이터를 포함하는 ZstdDict 인스턴스입니다. 함수 :func:`train_dict`를 사용하여 Zstandard 사전을 만들 수 있습니다.

compression.zstd.decompress(data, zstd_dict=None, options=None)¶

data (bytes-like object)를 압축 해제하여, 압축되지 않은 데이터를 bytes 객체로 반환합니다.

options 인자는 고급 압축 해제 매개변수를 포함하는 Python 딕셔너리입니다. 압축 매개변수에 대한 유효한 키와 값은 DecompressionParameter 문서의 일부로 설명되어 있습니다.

zstd_dict 인자는 압축 중에 사용된 훈련 데이터를 포함하는 ZstdDict 인스턴스입니다. 이는 반드시 압축 과정에서 사용된 Zstandard 사전과 일치해야 합니다.

data 가 여러 개의 개별 압축 프레임의 연결인 경우, 이 모든 프레임을 압축 해제하고 결과들의 연결을 반환합니다.

class compression.zstd.ZstdCompressor(level=None, options=None, zstd_dict=None)¶

압축 데이터를 증분적으로 사용할 수 있는 압축기 객체를 생성합니다.

단일 데이터 청크를 더 편리하게 압축하려면, 모듈 수준 함수 :func:`compress`를 참조하십시오.

level 인자는 압축 수준을 제어하는 정수입니다. level*은 *options*에서 :attr:`CompressionParameter.compression_level`을 설정하는 대안이 됩니다. *level*에 전달할 수 있는 값을 얻으려면 :attr:`~CompressionParameter.compression_level`에 대해 :meth:`~CompressionParameter.bounds`를 사용하십시오. 고급 압축 옵션이 필요한 경우, *level 인자를 생략하고 options 딕셔너리에서 CompressionParameter.compression_level 매개변수를 설정해야 합니다.

options 인자는 고급 압축 매개변수를 포함하는 Python 딕셔너리입니다. 압축 매개변수에 대한 유효한 키와 값은 CompressionParameter 문서의 일부로 설명되어 있습니다.

zstd_dict 인수는 압축 효율성을 개선하기 위해 훈련된 데이터를 포함하는 선택적 ZstdDict 인스턴스입니다. train_dict() 함수를 사용하여 Zstandard 사전을 생성할 수 있습니다.

compress(data, mode=ZstdCompressor.CONTINUE)¶

data\( bytes-like object)를 압축하여, 가능하다면 압축된 데이터가 포함된 bytes 객체와 함께 반환하거나, 그렇지 않으면 빈 bytes 객체를 반환합니다. data\의 일부는 나중에 compress()\ 및 flush()\ 호출에 사용하기 위해 내부적으로 버퍼링 될 수 있습니다. 반환된 데이터는 compress() 메서드에 대한 이전 호출의 출력과 연결되어야 합니다.

mode 인수는 ZstdCompressor 속성으로, CONTINUE, FLUSH_BLOCK, 또는 FLUSH_FRAME 중 하나입니다.

모든 데이터가 압축기에 제공되면, flush() 메서드를 호출하여 압축 공정을 마무리하십시오. 만약 FLUSH_FRAME 으로 mode 를 설정하고 compress() 가 호출된 경우, 빈 프레임을 새롭게 작성할 수 있으므로 반드시 flush() 는 호출해서는 안 됩니다.

flush(mode=ZstdCompressor.FLUSH_FRAME)¶

압축 공정을 완료하고 압축기 내부 버퍼에 저장된 모든 데이터를 포함하는 bytes 객체를 반환합니다.

mode 인수는 ZstdCompressor 속성으로, FLUSH_BLOCK 또는 FLUSH_FRAME 중 하나입니다.

set_pledged_input_size(size)¶

다음 프레임에 제공될 비압축 데이터 크기 size 를 지정합니다. CompressionParameter.content_size_flag 가 False 이거나 0 이 아닌 경우, 이 size 는 다음 프레임의 헤더에 기록됩니다. 크기가 0 인 경우 해당 프레임은 비어 있습니다. size 가 None 이면 프레임 헤더에는 프레임 크기가 생략됩니다. 비압축 데이터 크기를 포함하는 프레임은 특히 높은 압축 레벨에서 압축 해제 시 적은 메모리를 필요로 합니다.

If last_mode is not FLUSH_FRAME, a ValueError is raised as the compressor is not at the start of a frame. If the pledged size does not match the actual size of data provided to compress(), future calls to compress() or flush() may raise ZstdError and the last chunk of data may be lost.

:meth:`!set_pledged_input_size`가 다시 호출되지 않는 한, :meth:`flush`나 :meth:`~.compress`에 mode가 :attr:`FLUSH_FRAME`으로 전달된 후 다음 프레임은 헤더에 프레임 크기를 포함하지 않습니다.

CONTINUE¶

압축할 데이터를 더 많이 수집하며, 이 과정에서 즉시 출력이 발생하거나 아닐 수 있습니다. 이 모드는 블록 및 프레임당 데이터 양을 최대화하여 압축률을 최적화합니다.

FLUSH_BLOCK¶

데이터 스트림에 블록을 완전히 작성하고 기록합니다. 지금까지 반환된 데이터는 즉시 압축 해제될 수 있습니다. 이전 데이터는 compress() 호출로 생성되는 향후 블록에서 여전히 참조되어 압축률을 개선할 수 있습니다.

FLUSH_FRAME¶

프레임을 완전히 작성하고 기록합니다. :meth:`~.compress`에 제공되는 미래의 데이터는 새로운 프레임에 쓰여지며 과거 데이터를 참조할 수 없습니다.

last_mode¶

compress() 또는 CONTINUE(), FLUSH_BLOCK, 또는 FLUSH_FRAME 중 하나일 수 있습니다. 초기값은 :attr:`~.FLUSH_FRAME`이며, 이는 압축기가 새 프레임의 시작점에 있음을 나타냅니다.

class compression.zstd.ZstdDecompressor(zstd_dict=None, options=None)¶

데이터를 증분하여 압축 해제하는 데 사용할 수 있는 디컴프레서 객체를 만듭니다.

전체 압축 스트림을 한 번에 압축 해제하는 더 편리한 방법은 모듈 레벨 함수 :func:`decompress`를 참조하십시오.

options 인자는 고급 압축 해제 매개변수를 포함하는 Python 딕셔너리입니다. 압축 매개변수에 대한 유효한 키와 값은 DecompressionParameter 문서의 일부로 설명되어 있습니다.

zstd_dict 인자는 압축 중에 사용된 훈련 데이터를 포함하는 ZstdDict 인스턴스입니다. 이는 반드시 압축 과정에서 사용된 Zstandard 사전과 일치해야 합니다.

참고

This class does not transparently handle inputs containing multiple compressed frames, unlike the decompress() function and ZstdFile class. To decompress a multi-frame input, you should use decompress(), ZstdFile if working with a file object, or multiple ZstdDecompressor instances.

decompress(data, max_length=-1)¶

data\( bytes-like object)를 압축 해제하고, 압축되지 않은 데이터를 바이트열로 반환합니다. data\의 일부는 나중에 decompress()\를 호출할 때 사용하기 위해 내부적으로 버퍼링 될 수 있습니다. 반환된 데이터는 decompress() 메서드에 대한 이전 호출의 출력과 연결되어야 합니다.

max_length 가 음수가 아니면, 최대 max_length 바이트의 압축 해제된 데이터를 반환합니다. 이 한계에 도달하고 추가 출력을 생성할 수 있으면, needs_input 어트리뷰트가 False 로 설정됩니다. 이 경우, 다음 decompress() 호출은 data 를 b'' 로 제공하여 더 많은 출력을 얻을 수 있습니다. 따라서 전체 내용은 다음과 같이 읽을 수 있습니다:

process_output(d.decompress(data, max_length))
while not d.eof and not d.needs_input:
    process_output(d.decompress(b"", max_length))

모든 입력 데이터가 압축 해제되어 반환되었다면 (이는 max_length 바이트보다 적거나, 또는 max_length 가 음수였기 때문일 수 있음), needs_input 어트리뷰트는 True 로 설정됩니다.

프레임 끝 이후의 데이터를 압축 해제하려고 시도하면 ZstdError`가 발생합니다. 프레임 끝 이후에 발견되는 모든 데이터는 무시되고 :attr:`~.unused_data 어트리뷰트에 저장됩니다.

eof¶

스트림의 끝을 나타내는 마커에 도달했습니다면 True 입니다.

unused_data¶

압축 스트림 끝 이후 발견되는 데이터입니다.

스트림의 끝에 도달하기 전에는 이것이 b"" 일 것입니다.

needs_input¶

decompress() 메서드가 새로운 압축 입력을 요구하기 전에 더 많은 압축 해제된 데이터를 제공할 수 있으면 False 입니다.

Zstandard 딕셔너리¶

compression.zstd.train_dict(samples, dict_size)¶

Zstandard 딕셔너리를 학습시켜 ZstdDict 인스턴스를 반환합니다. Zstandard 딕셔너리는 반복성이 적어 압축하기 어려웠던 작은 크기의 데이터도 더 효율적으로 압축할 수 있게 해줍니다. 여러 유사한 그룹의 데이터를 압축하는 경우 (예: 유사한 파일), Zstandard 딕셔너리를 사용하면 압축률과 속도를 크게 향상시킬 수 있습니다.

samples 인수는 (bytes 객체의 이터러블)로, Zstandard 딕셔너리 학습에 사용되는 샘플 집합입니다.

dict_size 인수는 정수형으로, Zstandard 딕셔너리의 최대 크기(바이트 단위)입니다. Zstandard 문서는 절대 최대치가 100 KB를 초과하지 않아야 한다고 제안하지만, 실제 최대치는 데이터에 따라 더 작을 수 있습니다. 딕셔너리가 클수록 일반적으로 압축 속도는 느려지지만, 압축률은 향상됩니다. 딕셔너리가 작을수록 압축 속도는 빨라지지만, 압축률은 감소합니다.

compression.zstd.finalize_dict(zstd_dict, /, samples, dict_size, level)¶

“원시 콘텐츠(raw content)” Zstandard 딕셔너리를 일반 Zstandard 딕셔너리로 변환하기 위한 고급 함수입니다. “원시 콘텐츠” 딕셔너리는 일반적인 Zstandard 딕셔너리의 구조를 따를 필요가 없는 바이트 시퀀스입니다.

zstd_dict 인수는 dict_content`에 원시 딕셔너리 콘텐츠를 포함하는 :class:`ZstdDict 인스턴스입니다.

samples 인수는 (bytes 객체의 이터러블)으로, Zstandard 딕셔너리를 생성하는 데 사용되는 샘플 데이터를 포함합니다.

dict_size 인수는 정수형으로, Zstandard 딕셔너리의 최대 크기(바이트 단위)입니다. 최대 딕셔너리 크기에 대한 제안은 :func:`train_dict`를 참조하십시오.

level 인수는 (정수형) 이 딕셔너리를 사용하는 압축기에서 예상되는 압축 수준입니다. 딕셔너리 정보는 각 압축 수준마다 다르므로, 적절한 압축 수준에 맞게 조정하면 압축 효율성을 높일 수 있습니다.

class compression.zstd.ZstdDict(dict_content, /, *, is_raw=False)¶

Zstandard 딕셔너리의 래퍼입니다. 딕셔너리는 많은 작은 데이터 청크의 압축을 개선하는 데 사용될 수 있습니다. 샘플 데이터로부터 새 딕셔너리를 학습시켜야 하는 경우 :func:`train_dict`를 사용하십시오.

dict_content 인수는 (bytes-like object)으로, 이미 학습된 딕셔너리 정보입니다.

is_raw 인수는 부울 값이며, dict_content 의 의미를 제어하는 고급 매개변수입니다. True 는 dict_content 가 형식 제한이 없는 “원시 콘텐츠” 딕셔너리임을 의미합니다. False 는 train_dict() 나 외부 zstd CLI와 같은 Zstandard 함수로 생성된 일반적인 Zstandard 딕셔너리임을 의미합니다.

함수에 ZstdDict 를 전달할 때, as_digested_dict 및 as_undigested_dict 속성을 사용하여 zstd_dict 인수로 전달함으로써 딕셔너리가 로드되는 방식을 제어할 수 있습니다. 예를 들어, compress(data, zstd_dict=zd.as_digested_dict) 와 같습니다. 딕셔너리를 디제스팅하는 것은 Zstandard 딕셔너리를 로드하는 과정에서 발생하는 비용이 큰 작업입니다. 압축 또는 압축 해제를 여러 번 호출할 때 디제스티드 딕셔너리를 전달하면 딕셔너리 로딩 오버헤드를 줄일 수 있습니다.

디제스티드 딕셔너리

언디제스티드 딕셔너리

압축기 :keyword:`compressor`의 고급 매개변수이며, 딕셔너리의 매개변수에 의해 재정의될 수 있습니다.

window_log, hash_log, chain_log, search_log, min_match, target_length, strategy, enable_long_distance_matching, ldm_hash_log, ldm_min_match, ldm_bucket_size_log, ldm_hash_rate_log 및 일부 공개되지 않은 매개변수들입니다.

없음

:class:`!ZstdDict`는 내부적으로 딕셔너리를 캐싱합니다.

예. 동일한 압축 수준으로 디제스티드 딕셔너리를 다시 로드할 때 더 빠릅니다.

아니요. 언디제스티드 딕셔너리를 여러 번 로드하려면 컴프레서 객체를 재사용하는 것을 고려하세요.

만약 어떠한 속성 없이 :class:`!ZstdDict`를 전달하면, 압축 시 기본적으로 언디제스티드 딕셔너리가 사용되며 필요한 경우 디제스티드 딕셔너리가 생성되어 압축 해제 시 기본적으로 사용됩니다.

고급 매개변수 제어¶

class compression.zstd.CompressionParameter¶

데이터를 압축할 때 사용할 수 있는 고급 압축 매개변수 키를 포함하는 :class:`~enum.IntEnum`입니다.

bounds() 메서드는 모든 속성에 사용되어 해당 매개변수의 유효한 값을 가져올 수 있습니다.

매개변수는 선택 사항입니다. 생략된 모든 매개변수는 자동으로 값이 선택됩니다.

:attr:`~.compression_level`의 최솟값과 최댓값을 가져오는 예제:

lower, upper = CompressionParameter.compression_level.bounds()

:attr:`~.window_log`를 최대 크기로 설정하는 예제:

_lower, upper = CompressionParameter.window_log.bounds()
options = {CompressionParameter.window_log: upper}
compress(b'venezuelan beaver cheese', options=options)
bounds()¶

압축 매개변수의 정수 범위, (lower, upper) 튜플을 반환합니다. 이 메서드는 경계값을 검색하려는 속성에 호출해야 합니다. 예를 들어, compression_level 의 유효한 값을 얻으려면 CompressionParameter.compression_level.bounds() 의 결과를 확인할 수 있습니다.

최솟값과 최댓값 모두 포함적입니다.

compression_level¶

데이터 압축 속도와 비율에 영향을 미치는 다른 압축 매개변수를 설정하기 위한 고급 수단입니다.

일반적인 압축 수준은 0 보다 높습니다. 20 보다 큰 값은 “울트라” 압축으로 간주되며 다른 수준보다 더 많은 메모리가 필요합니다. 음수 값은 성능이 더 좋은 대신 오버헤드가 높은 압축 비율을 얻기 위해 사용할 수 있습니다.

레벨을 0으로 설정하면 :attr:`COMPRESSION_LEVEL_DEFAULT`가 사용됩니다.

window_log¶

압축기가 데이터를 압축할 때 사용할 수 있는 최대 허용 역참조 거리로, 2의 거듭제곱으로 표현되며 1 << window_log 바이트입니다. 이 매개변수는 압축의 메모리 사용량에 크게 영향을 미칩니다. 높은 값은 더 많은 메모리를 필요로 하지만 더 나은 압축 값을 얻습니다.

값이 0이면 자동으로 값이 선택됩니다.

hash_log¶

초기 프로브 테이블의 크기로, 2의 거듭제곱입니다. 결과 메모리 사용량은 1 << (hash_log+2) 바이트입니다. 더 큰 테이블은 전략 <= dfast 전략의 압축 속도를 개선합니다.

값이 0이면 자동으로 값이 선택됩니다.

chain_log¶

다중 프로브 검색 테이블의 크기이며, 2의 거듭제곱 단위입니다. 결과 메모리 사용량은 1 << (chain_log+2) 바이트입니다. 더 큰 테이블일수록 압축 성능이 좋아지고 속도는 느려집니다. 이 매개 변수는 fast 전략에서는 효과가 없습니다. 하지만 dfast 전략을 사용할 때도 유용하며, 이때는 보조 프로브 테이블을 정의합니다.

값이 0이면 자동으로 값이 선택됩니다.

search_log¶

검색 시도 횟수이며, 2의 거듭제곱 단위입니다. 시도 횟수가 많을수록 압축 성능이 좋아지고 속도는 느려집니다. 이 매개 변수는 fast 및 dfast 전략에서는 쓸모가 없습니다.

값이 0이면 자동으로 값이 선택됩니다.

min_match¶

검색된 일치 항목의 최소 크기입니다. 값이 클수록 압축 및 압축 해제 속도는 증가하지만 비율은 감소합니다. Zstandard는 여전히 더 작은 크기의 일치 항목을 찾을 수 있지만, 이 크기와 그보다 큰 크기를 찾도록 검색 알고리즘을 조정하는 것뿐입니다. 모든 전략이 < btopt 이전일 경우, 기본 최소값은 4 이고, 모든 전략이 > fast 인 경우, 유효 최대값은 6 입니다.

값이 0이면 자동으로 값이 선택됩니다.

target_length¶

이 필드의 영향은 선택한 :class:`Strategy`에 따라 달라집니다.

btopt, btultra 및 btultra2 전략의 경우, 이 값은 검색을 중지하는 데 “충분히 좋다”고 간주되는 일치 항목의 길이입니다. 값이 클수록 압축률이 좋아지지만, 압축 속도는 느려집니다.

fast 전략의 경우, 이는 매치 샘플링 사이의 거리입니다. 값이 클수록 압축은 빨라지지만, 압축률은 더 떨어집니다.

값이 0이면 자동으로 값이 선택됩니다.

strategy¶

선택한 전략 값이 높을수록 zstd가 사용하는 압축 기술이 더 복잡해져서, 압축률은 높아지고 압축 속도는 느려집니다.

더 보기

Strategy

enable_long_distance_matching¶

장거리 매칭은 더 먼 거리에서 큰 일치 항목을 찾는 방식으로 대용량 입회의 압축을 개선하는 데 사용될 수 있습니다. 이는 메모리 사용량과 윈도우 크기를 증가시킵니다.

True 또는 1 은 장거리 매칭을 활성화하고, False 또는 0 은 비활성화합니다.

이 매개 변수를 활성화하면 window_log 기본값을 128 MiB로 증가시키며, 명시적으로 다른 값으로 설정된 경우는 예외입니다. 이 설정은 window_log >= 128 MiB이고 압축 전략이 >= :attr:`~Strategy.btopt`(압축 레벨 16+)일 때 기본적으로 활성화됩니다.

ldm_hash_log¶

장거리 매칭용 테이블의 크기이며, 2의 거듭제곱 단위입니다. 값이 클수록 메모리 사용량과 압축률이 증가하지만, 압축 속도는 감소합니다.

값이 0이면 자동으로 값이 선택됩니다.

ldm_min_match¶

장거리 일치 검색기용 최소 매치 크기입니다. 너무 크거나 작은 값은 종종 압축률을 감소시킬 수 있습니다.

값이 0이면 자동으로 값이 선택됩니다.

ldm_bucket_size_log¶

충돌 해결를 위한 장거리 일치 검색기 해시 테이블의 각 버켓의 로그 크기입니다. 값이 클수록 충돌 해결가 개선되지만, 압축 속도는 느려집니다.

값이 0이면 자동으로 값이 선택됩니다.

ldm_hash_rate_log¶

장거리 일치 검색기 해시 테이블에 항목을 삽입/조회하는 빈도입니다. 값이 클수록 압축 속도가 개선됩니다. 기본값에서 크게 벗어나면 압축률 감소를 초래할 가능성이 높습니다.

값이 0이면 자동으로 값이 선택됩니다.

content_size_flag¶

압축 전에 알 수 있는 데이터의 크기를 Zstandard 프레임 헤더에 기록합니다.

이 플래그는 다음 시나리오에서만 적용됩니다:

  • 일회성 압축을 위해 :func:`compress`를 호출하는 경우

  • 프레임의 모든 데이터를 단일한 ZstdCompressor.compress() 호출로 제공하며, 이때 ZstdCompressor.FLUSH_FRAME 모드를 사용합니다.

  • Calling ZstdCompressor.set_pledged_input_size() with the exact amount of data that will be provided to the compressor prior to any calls to ZstdCompressor.compress() for the current frame. ZstdCompressor.set_pledged_input_size() must be called for each new frame.

다른 모든 압축 호출은 크기 정보를 프레임 헤더에 기록하지 않을 수 있습니다.

True 또는 1 은 콘텐츠 크기 플래그를 활성화하고, False 또는 0 은 비활성화합니다.

checksum_flag¶

압축되지 않은 콘텐츠에 대한 XXHash64을 사용한 4바이트 체크섬이 각 프레임 끝에 기록됩니다. Zstandard의 압축 해제 코드는 이 체크섬을 확인합니다. 불일치가 있으면 ZstdError 예외가 발생합니다.

True 또는 1 은 체크섬 생성을 활성화하고, False 또는 0 은 비활성화합니다.

dict_id_flag¶

:class:`ZstdDict`로 압축할 때, 딕셔너리 ID가 프레임 헤더에 기록됩니다.

True 또는 1 은 딕셔너리 ID 저장을 활성화하고, False 또는 0 은 비활성화합니다.

nb_workers¶

nb_workers 가 0보다 크면 병렬 압축을 위해 스레드 개수를 지정합니다. 이 경우 멀티스레드 압축이 활성화되며, 값 1 은 “단일 스레드 멀티스레드 모드”를 의미합니다. 작업자 수가 많으면 속도가 향상되지만, 메모리 사용량도 늘어나고 압축률은 약간 감소할 수 있습니다.

값이 0이면 멀티스레딩이 비활성화됩니다.

job_size¶

압축 작업의 크기(바이트). 이 값은 :attr:`~CompressionParameter.nb_workers`가 1 이상일 때만 적용됩니다. 각 압축 작업은 병렬로 완료되므로, 이 값이 활성화된 스레드 수에 간접적으로 영향을 미칠 수 있습니다.

값이 0이면 자동으로 값이 선택됩니다.

overlap_log¶

다음 작업을 위해 이전 작업(스레드)에서 다시 로드되는 데이터 양을 설정합니다. 이 값은 :attr:`~CompressionParameter.nb_workers`가 1 이상일 때만 사용됩니다. 허용 가능한 값의 범위는 0부터 9까지 다양합니다.

  • 0은 오버랩 양을 동적으로 설정함을 의미합니다.

  • 1은 오버랩이 없음을 의미합니다.

  • 9는 이전 작업에서 전체 창 크기를 사용함을 의미합니다.

각 증가는 오버랩 크기를 절반/두 배로 줄입니다. “8”은 window_size/2 의 오버랩을, “7”은 window_size/4 의 오버랩을 의미합니다.

class compression.zstd.DecompressionParameter¶

데이터를 압축 해제할 때 사용될 수 있는 고급 압축 해제 매개변수 키를 포함하는 :class:`~enum.IntEnum`입니다. 매개변수는 선택 사항이며, 생략된 매개변수는 자동으로 값이 선택됩니다.

bounds() 메서드는 모든 속성에 사용되어 해당 매개변수의 유효한 값을 가져올 수 있습니다.

:attr:`~.window_log_max`를 최대 크기로 설정하는 예시:

data = compress(b'Some very long buffer of bytes...')

_lower, upper = DecompressionParameter.window_log_max.bounds()

options = {DecompressionParameter.window_log_max: upper}
decompress(data, options=options)
bounds()¶

압축 해제 매개변수의 정수 경계값, (lower, upper) 튜플을 반환합니다. 이 메서드는 경계를 검색하려는 어트리뷰트에 호출해야 합니다.

최솟값과 최댓값 모두 포함적입니다.

window_log_max¶

압축 해제 중 사용되는 창의 최대 크기에 대한 밑이 2인 로그값입니다. 이는 데이터 압축 해제 시 사용되는 메모리 양을 제한하는 데 유용할 수 있습니다. 더 큰 최대 창 크기는 더 빠른 압축 해제를 가져옵니다.

값이 0이면 자동으로 값이 선택됩니다.

class compression.zstd.Strategy¶

compression.zstd.get_frame_info(frame_buffer)¶

Zstandard 프레임에 대한 메타데이터를 포함하는 FrameInfo 객체를 검색합니다. 프레임은 자신이 포함하는 압축 데이터와 관련된 메타데이터를 가집니다.

class compression.zstd.FrameInfo¶

Zstandard 프레임과 관련된 메타데이터입니다.

decompressed_size¶

프레임의 압축 해제된 내용 크기입니다.

dictionary_id¶

프레임을 압축 해제하는 데 필요한 Zstandard 딕셔너리 ID를 나타내는 정수입니다. 0 은 프레임 헤더에 딕셔너리 ID가 기록되지 않았음을 의미합니다. 이는 Zstandard 딕셔너리가 필요하지 않거나, 필요한 딕셔너리의 ID가 기록되지 않았을 수 있습니다.

compression.zstd.COMPRESSION_LEVEL_DEFAULT¶

Zstandard의 기본 압축 수준입니다: 3 입니다.

compression.zstd.zstd_version_info¶

런타임 zstd 라이브러리의 버전 번호 (major, minor, release)를 정수 튜플로 나타냅니다.

예시¶

압축 파일 읽기:

from compression import zstd

with zstd.open("file.zst") as f:
    file_content = f.read()

압축 파일 만들기:

from compression import zstd

data = b"Insert Data Here"
with zstd.open("file.zst", "w") as f:
    f.write(data)

메모리에서 데이터 압축하기:

from compression import zstd

data_in = b"Insert Data Here"
data_out = zstd.compress(data_in)

증분 압축:

from compression import zstd

comp = zstd.ZstdCompressor()
out1 = comp.compress(b"Some data\n")
out2 = comp.compress(b"Another piece of data\n")
out3 = comp.compress(b"Even more data\n")
out4 = comp.flush()
# 모든 부분 결과를 이어 붙입니다:
result = b"".join([out1, out2, out3, out4])

이미 열린 파일에 압축된 데이터 쓰기:

from compression import zstd

with open("myfile", "wb") as f:
    f.write(b"This data will not be compressed\n")
    with zstd.open(f, "w") as zstf:
        zstf.write(b"This *will* be compressed\n")
    f.write(b"Not compressed\n")

압축 매개변수를 사용하여 압축 파일 만들기:

from compression import zstd

options = {
   zstd.CompressionParameter.checksum_flag: 1
}
with zstd.open("file.zst", "w", options=options) as f:
    f.write(b"Mind if I squeeze in?")
